# Multipart Upload

Use multipart upload for files larger than 100 MB. The `multipartUpload` method handles part management, parallel uploading, and checkpoint-based resume automatically.

## Resumable multipart upload

```javascript title="multipartUpload.js"
const path = require('path');

const result = await client.multipartUpload(
  'large-files/video.mp4',
  path.normalize('/path/to/video.mp4'),
  {
    partSize: 1024 * 1024,  // 1MB per part
    parallel: 4,             // 4 parallel uploads
    progress: (percentage, checkpoint) => {
      console.log(`Progress: ${(percentage * 100).toFixed(1)}%`);
      // Save checkpoint for resume
      // fs.writeFileSync('checkpoint.json', JSON.stringify(checkpoint));
    },
  }
);
console.log('Upload complete:', result.name);
```

## Resume an interrupted upload

```javascript title="resumeUpload.js"
const fs = require('fs');

let checkpoint;
try {
  checkpoint = JSON.parse(fs.readFileSync('checkpoint.json', 'utf8'));
} catch (e) {
  // No checkpoint, start fresh
}

const result = await client.multipartUpload(
  'large-files/video.mp4',
  '/path/to/video.mp4',
  {
    checkpoint,  // Resume from checkpoint
    progress: (percentage, cp) => {
      fs.writeFileSync('checkpoint.json', JSON.stringify(cp));
    },
  }
);
```

## Cancel a multipart upload

```javascript title="abortMultipart.js"
await client.abortMultipartUpload('large-files/video.mp4', uploadId);
```

## List multipart uploads

```javascript title="listMultipart.js"
const result = await client.listUploads({ prefix: 'large-files/' });
result.uploads.forEach(upload => {
  console.log('Key:', upload.name, 'UploadId:', upload.uploadId);
});
```
