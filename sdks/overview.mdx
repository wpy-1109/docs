# SDK Overview

Alibaba Cloud OSS provides official SDKs for all major programming languages and platforms. Each SDK wraps the OSS REST API with idiomatic language patterns, handles authentication, retries, and data integrity checks automatically.

## Choosing a version

Most SDKs offer two versions:

<CardGroup cols={2}>
  <Card title="V2 SDK (Recommended)" icon="star">
    Latest architecture with V4 signatures, builder patterns, credential provider chains, and enhanced features. Available for Python, Java, Go, PHP, and .NET.
  </Card>
  <Card title="V1 SDK (Stable)" icon="shield-check">
    Production-proven and widely deployed. Full feature support. Use if V2 is not yet available for your language or you need maximum stability.
  </Card>
</CardGroup>

## Common credential methods

All SDKs support the following authentication methods (in order of preference):

1. **Environment variables** -- Set `OSS_ACCESS_KEY_ID` and `OSS_ACCESS_KEY_SECRET`
2. **STS temporary credentials** -- Short-lived tokens from the Security Token Service
3. **Instance RAM role** -- Automatic credentials on ECS/Container Service instances
4. **Static credentials** -- Direct AccessKey ID and Secret (not recommended for production)

<Warning>
Never hardcode AccessKey credentials in source code. Use environment variables or STS temporary credentials for production deployments.
</Warning>
