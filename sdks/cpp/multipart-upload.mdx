# Multipart upload

Multipart upload splits large files into smaller parts for parallel uploading. This is recommended for files larger than 100 MB.

## Multipart upload flow

<Steps>
  <Step title="Initiate">
    Call `InitiateMultipartUpload` to get an `UploadId`.
  </Step>
  <Step title="Upload Parts">
    Split the file and upload each part with `UploadPart`.
  </Step>
  <Step title="Complete">
    Call `CompleteMultipartUpload` to assemble all parts.
  </Step>
</Steps>

## Complete example

```cpp title="multipartUpload.cpp"
#include <alibabacloud/oss/OssClient.h>
#include <fstream>
using namespace AlibabaCloud::OSS;

// ... (initialization) ...

std::string BucketName = "my-bucket";
std::string ObjectName = "large-file.zip";
std::string FilePath = "/path/to/large-file.zip";
int64_t PartSize = 5 * 1024 * 1024; // 5 MB per part

// Step 1: Initiate multipart upload
InitiateMultipartUploadRequest initRequest(BucketName, ObjectName);
auto initOutcome = client.InitiateMultipartUpload(initRequest);
std::string uploadId = initOutcome.result().UploadId();

// Step 2: Upload parts
std::fstream file(FilePath, std::ios::in | std::ios::binary);
file.seekg(0, std::ios::end);
int64_t fileSize = file.tellg();
file.seekg(0, std::ios::beg);

PartList partList;
int partNumber = 1;

for (int64_t offset = 0; offset < fileSize; offset += PartSize) {
    int64_t currentPartSize = std::min(PartSize, fileSize - offset);
    auto content = std::make_shared<std::stringstream>();
    std::vector<char> buffer(currentPartSize);
    file.read(buffer.data(), currentPartSize);
    content->write(buffer.data(), currentPartSize);

    UploadPartRequest request(BucketName, ObjectName, content);
    request.setUploadId(uploadId);
    request.setPartNumber(partNumber);
    request.setContentLength(currentPartSize);

    auto uploadOutcome = client.UploadPart(request);
    Part part(partNumber, uploadOutcome.result().ETag());
    partList.push_back(part);

    std::cout << "Part " << partNumber << " uploaded (" << currentPartSize << " bytes)" << std::endl;
    partNumber++;
}
file.close();

// Step 3: Complete multipart upload
CompleteMultipartUploadRequest completeRequest(BucketName, ObjectName);
completeRequest.setUploadId(uploadId);
completeRequest.setPartList(partList);

auto completeOutcome = client.CompleteMultipartUpload(completeRequest);

if (completeOutcome.isSuccess()) {
    std::cout << "Multipart upload complete." << std::endl;
} else {
    std::cout << "Error: " << completeOutcome.error().Message() << std::endl;
}
```

## Abort a multipart upload

Cancel an in-progress multipart upload and clean up uploaded parts:

```cpp title="abortUpload.cpp"
AbortMultipartUploadRequest request("my-bucket", "large-file.zip", uploadId);
client.AbortMultipartUpload(request);
```

## List multipart uploads

```cpp title="listUploads.cpp"
ListMultipartUploadsRequest request("my-bucket");
auto outcome = client.ListMultipartUploads(request);

if (outcome.isSuccess()) {
    for (const auto& upload : outcome.result().MultipartUploadList()) {
        std::cout << "Key: " << upload.Key << ", Upload ID: " << upload.UploadId << std::endl;
    }
}
```

## List uploaded parts

```cpp title="listParts.cpp"
ListPartsRequest request("my-bucket", "large-file.zip");
request.setUploadId(uploadId);

auto outcome = client.ListParts(request);

if (outcome.isSuccess()) {
    for (const auto& part : outcome.result().PartList()) {
        std::cout << "Part: " << part.PartNumber() << ", Size: " << part.Size() << std::endl;
    }
}
```

<Tip>
  For a simpler API, use `ResumableUploadObject` which handles multipart upload, parallel threads, and checkpoint recovery automatically.
</Tip>
