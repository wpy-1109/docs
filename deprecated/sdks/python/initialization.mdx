# Initialization

Before performing any OSS operations, you need to create and configure an OSS client with valid credentials.

## V2 SDK

### Using environment variables (Recommended)

Set the following environment variables before running your application:

```bash
export OSS_ACCESS_KEY_ID="your-access-key-id"
export OSS_ACCESS_KEY_SECRET="your-access-key-secret"
```

```python title="init_v2.py"
import alibabacloud_oss_v2 as oss

credentials_provider = oss.credentials.EnvironmentVariableCredentialsProvider()

cfg = oss.config.load_default()
cfg.credentials_provider = credentials_provider
cfg.region = 'cn-hangzhou'

client = oss.Client(cfg)
```

### Using static credentials

```python title="init_v2_static.py"
import alibabacloud_oss_v2 as oss

credentials_provider = oss.credentials.StaticCredentialsProvider(
    access_key_id='your-access-key-id',
    access_key_secret='your-access-key-secret'
)

cfg = oss.config.load_default()
cfg.credentials_provider = credentials_provider
cfg.region = 'cn-hangzhou'

client = oss.Client(cfg)
```

<Warning>
Never hardcode credentials in production code. Use environment variables, STS temporary credentials, or instance RAM roles.
</Warning>

## V1 SDK

### Using environment variables (Recommended)

```python title="init_v1.py"
import oss2
from oss2.credentials import EnvironmentVariableCredentialsProvider

# Reads OSS_ACCESS_KEY_ID and OSS_ACCESS_KEY_SECRET from environment
auth = oss2.ProviderAuthV4(EnvironmentVariableCredentialsProvider())
bucket = oss2.Bucket(auth, 'https://oss-cn-hangzhou.aliyuncs.com', 'your-bucket-name')
```

### Using accessKey directly

```python title="init_v1_direct.py"
import oss2

auth = oss2.Auth('your-access-key-id', 'your-access-key-secret')
bucket = oss2.Bucket(auth, 'https://oss-cn-hangzhou.aliyuncs.com', 'your-bucket-name')
```

### Using STS temporary credentials

```python title="init_v1_sts.py"
import oss2

auth = oss2.StsAuth(
    'temporary-access-key-id',
    'temporary-access-key-secret',
    'security-token'
)
bucket = oss2.Bucket(auth, 'https://oss-cn-hangzhou.aliyuncs.com', 'your-bucket-name')
```

## Credential methods

Listed in order of preference for production use:

| Method | Security | Use Case |
|--------|----------|----------|
| Environment variables | High | CI/CD, containers, local development |
| RAM role / Instance role | Highest | ECS instances, Container Service |
| STS temporary credentials | High | Temporary, scoped access |
| Static credentials | Low | Testing only |

## Configuration options

| Option | Description | Default |
|--------|-------------|---------|
| `region` | Region identifier (e.g., `cn-hangzhou`) | Required |
| `endpoint` | Custom OSS endpoint | Auto-generated from region |
| `credentials_provider` | Credential provider | Required |
| `connect_timeout` | Connection timeout (seconds) | 60 |
| `readwrite_timeout` | Read/write timeout (seconds) | 120 |
