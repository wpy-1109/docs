## URL structure

An OSS access URL follows this pattern:

```
https://<BucketName>.<Endpoint>/<ObjectKey>
```

For example: `https://examplebucket.oss-cn-hangzhou.aliyuncs.com/myfile.txt`

## Core concepts

| Concept | Description | Example |
|---------|-------------|---------|
| Region ID | General region identifier | `cn-hangzhou` |
| OSS Region ID | OSS-specific region identifier | `oss-cn-hangzhou` |
| Endpoint | Service access address | `oss-cn-hangzhou.aliyuncs.com` |
| Bucket domain | Full access URL for a bucket | `examplebucket.oss-cn-hangzhou.aliyuncs.com` |

## Endpoint types

| Type | Format | Use Case |
|------|--------|----------|
| Public | `oss-<region>.aliyuncs.com` | Access from the internet |
| Internal | `oss-<region>-internal.aliyuncs.com` | Access from ECS/services in the same region (no traffic fees) |
| Acceleration | `oss-accelerate.aliyuncs.com` | Cross-region acceleration |
| Dual-stack | `oss-<region>.aliyuncs.com` | IPv4 and IPv6 access |

<Tip>
Use the internal endpoint when accessing OSS from Alibaba Cloud services (ECS, Function Compute) in the same region. Internal traffic is free and has lower latency.
</Tip>

## Addressing styles

| Style | URL Format | Description |
|-------|-----------|-------------|
| Virtual-hosted (default) | `https://bucket.endpoint/key` | Bucket name is in the hostname |
| Path-style | `https://endpoint/bucket/key` | Bucket name is in the URL path. Required for PrivateLink. |
