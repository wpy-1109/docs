When you no longer need a bucket, delete it to stop billing. OSS charges are primarily for resources within a bucket. Before you delete a bucket, you must clear all of its contents.

<Warning>
Deleting a bucket permanently removes all data, including objects, previous versions, and parts. This operation is irreversible. Back up your data before proceeding.
</Warning>

## Prerequisites

- SDK installed and configured ([quickstart](/quickstart))
- An Alibaba Cloud account with OSS activated
- An AccessKey pair or STS temporary credentials
- The `oss:DeleteBucket` permission (plus `oss:ListObjects` and `oss:DeleteObject` for cleanup)

## Risk assessment

Before deleting a bucket, consider the following risks:

| Risk | Description |
|------|-------------|
| **Data loss** | All files, versions, and parts are permanently removed. |
| **Business interruption** | Websites, apps, or CDN services that depend on the bucket will stop working. |
| **Name squatting** | After deletion, the bucket name enters a 4--8 hour cool-down period before it becomes available again. |
| **Replication impact** | If cross-region or same-region replication is enabled, the delete operation may also affect the destination bucket. |

## Step 1: clear all resources

A bucket must be empty before it can be deleted. You must remove all:
- **Objects** (current and previous versions)
- **Parts** (incomplete multipart uploads)
- **Access Points**
- **Accelerators**
- **RTMP Ingest URLs (LiveChannels)**

<Tabs>
  <Tab title="Console">
    Use the **Delete Bucket** wizard in the OSS console. Navigate to your bucket, click **Delete Bucket** in the left navigation pane, and follow the prompts to clear each resource type.
  </Tab>
  <Tab title="ossutil CLI">
    ```bash
    # Delete all objects (including all versions)
    ossutil rm oss://examplebucket/ -r

    # Delete all incomplete multipart upload parts
    ossutil rm oss://examplebucket -m -r -f
    ```
  </Tab>
</Tabs>

<Tip>
For buckets with millions of objects, use [lifecycle rules](/guides/data-management/lifecycle-rules) to automatically delete all objects. Configure a rule to expire objects after 1 day, and OSS will clean them up automatically.
</Tip>

## Step 2: delete the bucket

<Tabs>
  <Tab title="Console">
    Navigate to the **Delete Bucket** page, click **Delete Bucket**, then confirm by clicking **Delete** and **OK** in the dialog.
  </Tab>
  <Tab title="ossutil CLI">
    ```bash
    ossutil api delete-bucket --bucket examplebucket
    ```
  </Tab>
</Tabs>

## Using an SDK

<Tabs>
  <Tab title="Python">
    ```python title="delete_bucket.py"
    import alibabacloud_oss_v2 as oss

    credentials_provider = oss.credentials.EnvironmentVariableCredentialsProvider()
    cfg = oss.config.load_default()
    cfg.credentials_provider = credentials_provider
    cfg.region = "cn-hangzhou"

    client = oss.Client(cfg)

    result = client.delete_bucket(oss.DeleteBucketRequest(
        bucket="examplebucket"
    ))

    print(f"Status: {result.status_code}, Request ID: {result.request_id}")
    ```
  </Tab>
  <Tab title="Java">
    ```java title="DeleteBucket.java"
    import com.aliyun.oss.*;
    import com.aliyun.oss.common.auth.*;
    import com.aliyun.oss.common.comm.SignVersion;

    public class DeleteBucket {
        public static void main(String[] args) throws Exception {
            String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
            String region = "cn-hangzhou";
            EnvironmentVariableCredentialsProvider credentialsProvider =
                CredentialsProviderFactory.newEnvironmentVariableCredentialsProvider();

            ClientBuilderConfiguration clientConfig = new ClientBuilderConfiguration();
            clientConfig.setSignatureVersion(SignVersion.V4);
            OSS ossClient = OSSClientBuilder.create()
                .endpoint(endpoint)
                .credentialsProvider(credentialsProvider)
                .clientConfiguration(clientConfig)
                .region(region)
                .build();

            try {
                ossClient.deleteBucket("examplebucket");
                System.out.println("Bucket deleted successfully.");
            } finally {
                ossClient.shutdown();
            }
        }
    }
    ```
  </Tab>
  <Tab title="Go">
    ```go title="delete_bucket.go"
    package main

    import (
        "context"
        "log"
        "github.com/aliyun/alibabacloud-oss-go-sdk-v2/oss"
        "github.com/aliyun/alibabacloud-oss-go-sdk-v2/oss/credentials"
    )

    func main() {
        cfg := oss.LoadDefaultConfig().
            WithCredentialsProvider(credentials.NewEnvironmentVariableCredentialsProvider()).
            WithRegion("cn-hangzhou")

        client := oss.NewClient(cfg)

        result, err := client.DeleteBucket(context.TODO(), &oss.DeleteBucketRequest{
            Bucket: oss.Ptr("examplebucket"),
        })
        if err != nil {
            log.Fatalf("Failed to delete bucket: %v", err)
        }

        log.Printf("Bucket deleted: %#v\n", result)
    }
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript title="deleteBucket.js"
    const OSS = require('ali-oss');

    const client = new OSS({
      region: 'oss-cn-hangzhou',
      accessKeyId: process.env.OSS_ACCESS_KEY_ID,
      accessKeySecret: process.env.OSS_ACCESS_KEY_SECRET,
      authorizationV4: true,

    async function deleteBucket() {
      try {
        const result = await client.deleteBucket('examplebucket');
        console.log('Bucket deleted:', result);
      } catch (err) {
        console.error('Error:', err);
      }
    }

    deleteBucket();
    ```

## Error handling

| Error Code | Cause | Solution |
|------------|-------|----------|
| `BucketNotEmpty` | The bucket still contains resources. | Clear all objects, parts, and access points, then retry. |
| `AccessDenied` | Insufficient permissions. | Ensure your RAM identity has `AliyunOSSFullAccess` or the required delete permissions. |
| `NoSuchBucket` | The bucket does not exist. | Verify the bucket name is correct. |

## FAQ

<AccordionGroup>
  <Accordion title="Why can't I create a bucket with the same name immediately after deletion?">
    Deleted bucket names enter a cool-down period of 4--8 hours. During this period, no user can create a bucket with the same name. After the cool-down, the name becomes available to all users.
  </Accordion>
  <Accordion title="How do I completely stop OSS billing?">
    Delete all buckets under your account. As long as a bucket exists, even if empty, you may still incur charges.
  </Accordion>
  <Accordion title="Can I recover a bucket after accidental deletion?">
    No. Bucket deletion is irreversible. Always back up your data before deleting.
  </Accordion>
</AccordionGroup>
