---
title: 'PutBucket'
description: 'Create a new bucket'
---

Creates a new bucket. Bucket names are globally unique across all OSS users.

## Request

```
PUT / HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
```

### Request body (Optional)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CreateBucketConfiguration>
    <StorageClass>Standard</StorageClass>
    <DataRedundancyType>LRS</DataRedundancyType>
</CreateBucketConfiguration>
```

### Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `x-oss-acl` | String | No | Bucket ACL. Values: `private` (default), `public-read`, `public-read-write`. |
| `x-oss-resource-group-id` | String | No | Resource group ID to assign the bucket to. |

### Body elements

| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `StorageClass` | String | No | Default storage class. Values: `Standard` (default), `IA`, `Archive`, `ColdArchive`, `DeepColdArchive`. |
| `DataRedundancyType` | String | No | Data redundancy type. Values: `LRS` (default, locally redundant), `ZRS` (zone-redundant). |

## Response

**Success:** HTTP 200 with empty body.

```http
HTTP/1.1 200 OK
x-oss-request-id: 534B371674E88A4D8906****
Location: /my-bucket
```

## Examples

### Create a standard bucket

```http
PUT / HTTP/1.1
Host: my-bucket.oss-cn-hangzhou.aliyuncs.com
Date: Thu, 17 Apr 2025 03:15:40 GMT
x-oss-acl: private
Authorization: OSS4-HMAC-SHA256 Credential=LTAI****/20250417/cn-hangzhou/oss/aliyun_v4_request,Signature=****

<?xml version="1.0" encoding="UTF-8"?>
<CreateBucketConfiguration>
    <StorageClass>Standard</StorageClass>
    <DataRedundancyType>LRS</DataRedundancyType>
</CreateBucketConfiguration>
```

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `BucketAlreadyExists` | 409 | A bucket with this name already exists. |
| `TooManyBuckets` | 400 | Account has reached the maximum number of buckets (default: 100). |
| `InvalidBucketName` | 400 | Bucket name violates naming rules. |

## Naming rules

- 3 to 63 characters long
- Lowercase letters, numbers, and hyphens only
- Must start and end with a lowercase letter or number
- Cannot start with `http://` or `https://`
