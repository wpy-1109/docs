## Common request headers

These headers can be used in all OSS API requests.

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `Authorization` | String | Yes (except anonymous) | Authentication information. See [Signature V4](/api-reference/signature-v4). |
| `Content-Length` | String | Conditional | Size of the HTTP message body in bytes. Required for PUT operations without chunked encoding. |
| `Content-Type` | String | No | MIME type of the request body (e.g., `application/xml`, `application/octet-stream`). |
| `Content-MD5` | String | No | Base64-encoded 128-bit MD5 digest of the message body for integrity verification. |
| `Date` | String | Yes | GMT date/time of the request per RFC 2616 (e.g., `Thu, 17 Apr 2025 03:15:40 GMT`). |
| `Host` | String | Yes | Target endpoint (e.g., `mybucket.oss-cn-hangzhou.aliyuncs.com`). |
| `x-oss-date` | String | No | ISO 8601 UTC timestamp (e.g., `20250417T031540Z`). Overrides `Date` for signature calculation. |
| `x-oss-content-sha256` | String | Yes (V4) | SHA-256 hash of the payload. Set to `UNSIGNED-PAYLOAD` for unsigned payloads. |
| `x-oss-security-token` | String | Conditional | STS security token. Required when using temporary credentials from STS. |
| `x-oss-request-payer` | String | No | Set to `requester` when accessing a pay-by-requester bucket. |

## Common response headers

These headers are returned in all OSS API responses.

| Header | Type | Description |
|--------|------|-------------|
| `Content-Length` | String | Size of the response body in bytes. |
| `Content-Type` | String | MIME type of the response body (typically `application/xml`). |
| `Connection` | String | Connection status (`keep-alive` or `close`). |
| `Date` | String | GMT date/time of the response. |
| `ETag` | String | Entity tag for the object. For objects uploaded via PutObject, this is the MD5 hash. For multipart uploads, it has the format `{MD5}-{PartCount}`. |
| `Server` | String | Always `AliyunOSS`. |
| `x-oss-request-id` | String | Unique identifier for the request. Include this in support tickets. |
| `x-oss-server-time` | String | Server processing time in milliseconds. |
| `x-oss-ec` | String | Error code identifier for detailed troubleshooting. |

## Content-Type values

Common MIME types used with OSS:

| File Type | Content-Type |
|-----------|-------------|
| JSON | `application/json` |
| XML | `application/xml` |
| Plain text | `text/plain` |
| HTML | `text/html` |
| JPEG image | `image/jpeg` |
| PNG image | `image/png` |
| PDF | `application/pdf` |
| Binary/stream | `application/octet-stream` |
| Gzip | `application/gzip` |

<Note>
If you do not set `Content-Type`, OSS infers it from the file extension. If the extension is not recognized, it defaults to `application/octet-stream`.
</Note>
