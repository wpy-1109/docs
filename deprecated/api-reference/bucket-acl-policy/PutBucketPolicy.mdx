---
title: 'PutBucketPolicy'
description: 'Set a bucket policy for resource-based access control'
---

Configures a bucket policy. Bucket policies provide resource-based access control, allowing you to grant cross-account access and set conditions like IP restrictions.

## Request

```
PUT /?policy HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
Content-Type: application/json
```

### Request body

The request body is a JSON policy document.

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:GetObject"],
      "Principal": ["*"],
      "Resource": ["acs:oss:*:*:my-bucket/*"],
      "Condition": {
        "IpAddress": {
          "acs:SourceIp": ["192.168.0.0/16"]
        }
      }
    }
  ]
}
```

### Policy elements

| Element | Description |
|---------|-------------|
| `Version` | Policy version. Always `"1"`. |
| `Statement` | Array of permission statements. |
| `Effect` | `Allow` or `Deny`. |
| `Action` | OSS actions (e.g., `oss:GetObject`, `oss:PutObject`, `oss:*`). |
| `Principal` | Alibaba Cloud account IDs or `*` for all. |
| `Resource` | ARN of the bucket/objects (e.g., `acs:oss:*:*:my-bucket/*`). |
| `Condition` | Optional conditions like IP restrictions, TLS version, etc. |

## Response

**Success:** HTTP 200 with empty body.

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `MalformedPolicy` | 400 | The policy JSON is invalid. |
| `NoSuchBucket` | 404 | The bucket does not exist. |
| `AccessDenied` | 403 | You do not have permission to set the policy. |
