---
title: 'PutObject'
description: 'Upload an object to a bucket'
---

Uploads an object to a bucket. The maximum size for a single PutObject request is 5 GB. For larger files, use [Multipart Upload](/api-reference/multipart-upload/InitiateMultipartUpload).

## Request

```
PUT /{ObjectName} HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
Content-Type: {ContentType}
Content-Length: {ContentLength}
```

### Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `Content-Type` | String | No | MIME type of the object. Auto-detected from extension if omitted. |
| `Content-Length` | String | No | Size of the body in bytes. |
| `Content-MD5` | String | No | Base64 MD5 hash for integrity verification. |
| `Content-Disposition` | String | No | `inline` (display in browser) or `attachment;filename="name"` (download). |
| `Content-Encoding` | String | No | Content encoding (`gzip`, `deflate`, `br`). |
| `Cache-Control` | String | No | Caching behavior (`no-cache`, `max-age=3600`). |
| `Expires` | String | No | Expiration time per RFC 2616. |
| `x-oss-storage-class` | String | No | Storage class: `Standard`, `IA`, `Archive`, `ColdArchive`, `DeepColdArchive`. |
| `x-oss-object-acl` | String | No | Object ACL: `default`, `private`, `public-read`, `public-read-write`. |
| `x-oss-server-side-encryption` | String | No | Encryption: `AES256`, `KMS`. |
| `x-oss-server-side-encryption-key-id` | String | No | KMS CMK ID (when using KMS). |
| `x-oss-forbid-overwrite` | String | No | Set to `true` to prevent overwriting existing objects. |
| `x-oss-tagging` | String | No | Object tags as URL-encoded key-value pairs (e.g., `TagA=A&TagB=B`). |
| `x-oss-meta-*` | String | No | User-defined metadata (total max 8 KB). |

### Request body

The object data (binary content).

## Response

```http
HTTP/1.1 200 OK
ETag: "D41D8CD98F00B204E9800998ECF8****"
Content-MD5: 1B2M2Y8AsgTpgAmY7PhC****
x-oss-hash-crc64ecma: 316181249502703****
x-oss-request-id: 5C06A3B67B8B5A3DA422****
```

### Response headers

| Header | Type | Description |
|--------|------|-------------|
| `ETag` | String | MD5 hash of the uploaded object. |
| `Content-MD5` | String | MD5 hash of the uploaded content. |
| `x-oss-hash-crc64ecma` | String | CRC-64 checksum. |
| `x-oss-version-id` | String | Version ID (only when versioning is enabled). |

## Examples

### Simple upload

<CodeGroup>
```http Request
PUT /test.txt HTTP/1.1
Host: my-bucket.oss-cn-hangzhou.aliyuncs.com
Content-Type: text/plain
Date: Thu, 17 Apr 2025 03:15:40 GMT
Authorization: OSS4-HMAC-SHA256 Credential=LTAI****/20250417/cn-hangzhou/oss/aliyun_v4_request,Signature=****

Hello, OSS!
```

```http Response
HTTP/1.1 200 OK
ETag: "D41D8CD98F00B204E9800998ECF8****"
x-oss-hash-crc64ecma: 316181249502703****
x-oss-request-id: 5C06A3B67B8B5A3DA422****
```
</CodeGroup>

### Upload with storage class and encryption

```http
PUT /archive-data.bin HTTP/1.1
Host: my-bucket.oss-cn-hangzhou.aliyuncs.com
Content-Type: application/octet-stream
Content-Length: 344606
x-oss-storage-class: Archive
x-oss-server-side-encryption: AES256
Authorization: OSS4-HMAC-SHA256 Credential=LTAI****/20250417/cn-hangzhou/oss/aliyun_v4_request,Signature=****

[344606 bytes of object data]
```

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `MissingContentLength` | 411 | Missing `Content-Length` without chunked encoding. |
| `InvalidEncryptionAlgorithmError` | 400 | Invalid `x-oss-server-side-encryption` value. |
| `AccessDenied` | 403 | Insufficient permissions. |
| `NoSuchBucket` | 404 | The bucket does not exist. |
| `InvalidObjectName` | 400 | Object name is invalid or exceeds limits. |
| `InvalidArgument` | 400 | Invalid parameter value (e.g., object exceeds 5 GB or invalid `x-oss-storage-class`). |
| `RequestTimeout` | 400 | Body not sent within timeout. |
| `FileAlreadyExists` | 409 | Object exists and `x-oss-forbid-overwrite` is `true`. |
