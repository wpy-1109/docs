---
title: 'OptionObject'
description: 'Send a CORS preflight request'
---

Sends a preflight (OPTIONS) request to check whether the actual cross-origin request will be allowed.

## Request

```
OPTIONS /ObjectName HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
Origin: https://example.com
Access-Control-Request-Method: PUT
Access-Control-Request-Headers: x-oss-meta-custom
```

### Required headers

| Header | Type | Description |
|--------|------|-------------|
| `Origin` | String | Origin of the request. |
| `Access-Control-Request-Method` | String | HTTP method for the actual request. |
| `Access-Control-Request-Headers` | String | Headers for the actual request (comma-separated). |

## Response

```http
HTTP/1.1 200 OK
Access-Control-Allow-Origin: https://example.com
Access-Control-Allow-Methods: GET, PUT
Access-Control-Allow-Headers: x-oss-meta-custom
Access-Control-Max-Age: 3600
```

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `AccessForbidden` | 403 | CORS rules do not allow this origin/method/header combination. |
