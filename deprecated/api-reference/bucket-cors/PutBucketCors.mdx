---
title: 'PutBucketCors'
description: 'Configure CORS rules for a bucket'
---


## Request

```
PUT /?cors HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
```

### Request body

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CORSConfiguration>
  <CORSRule>
    <AllowedOrigin>https://example.com</AllowedOrigin>
    <AllowedMethod>GET</AllowedMethod>
    <AllowedMethod>PUT</AllowedMethod>
    <AllowedHeader>*</AllowedHeader>
    <ExposeHeader>x-oss-request-id</ExposeHeader>
    <MaxAgeSeconds>3600</MaxAgeSeconds>
  </CORSRule>
</CORSConfiguration>
```

| Element | Type | Description |
|---------|------|-------------|
| `AllowedOrigin` | String | Allowed origin. Use `*` for all origins. |
| `AllowedMethod` | String | Allowed HTTP method: `GET`, `PUT`, `POST`, `DELETE`, `HEAD`. |
| `AllowedHeader` | String | Allowed request headers. Use `*` for all. |
| `ExposeHeader` | String | Response headers exposed to the browser. |
| `MaxAgeSeconds` | Integer | Preflight cache duration in seconds. |

## Response

**Success:** HTTP 200 with empty body.
