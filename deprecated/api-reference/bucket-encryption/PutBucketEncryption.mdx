---
title: 'PutBucketEncryption'
description: 'Configure server-side encryption for a bucket'
---

Configures the default server-side encryption rules for a bucket. Objects uploaded without explicit encryption headers will use the bucket default.

## Request

```
PUT /?encryption HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
```

### Request body

```xml
<?xml version="1.0" encoding="UTF-8"?>
<ServerSideEncryptionRule>
  <ApplyServerSideEncryptionByDefault>
    <SSEAlgorithm>KMS</SSEAlgorithm>
    <KMSDataEncryption>SM4</KMSDataEncryption>
    <KMSMasterKeyID>9468da86-3509-4f8d-a61e-****</KMSMasterKeyID>
  </ApplyServerSideEncryptionByDefault>
</ServerSideEncryptionRule>
```

| Element | Type | Required | Description |
|---------|------|----------|-------------|
| `SSEAlgorithm` | String | Yes | Encryption algorithm: `KMS`, `AES256`, or `SM4`. |
| `KMSDataEncryption` | String | No | KMS data encryption algorithm. Set to `SM4` for SM4 encryption. |
| `KMSMasterKeyID` | String | No | KMS CMK ID. If omitted, OSS uses the default service key. |

## Response

**Success:** HTTP 200 with empty body.

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `NoSuchBucket` | 404 | The bucket does not exist. |
| `InvalidEncryptionAlgorithmError` | 400 | Invalid encryption algorithm. |
