<AccordionGroup>
  <Accordion title="Which SDK should I use?">
    Choose the SDK for your programming language:
    - **Java**: `com.aliyun.oss:aliyun-sdk-oss`
    - **Python**: `pip install oss2`
    - **Go**: `github.com/aliyun/alibabacloud-oss-go-sdk-v2`
    - **Node.js**: `npm install ali-oss`
    - **PHP**: `composer require aliyuncs/oss-sdk-php`
    - **.NET**: `NuGet: Aliyun.OSS.SDK.NetCore`
    - **C++**: Build from source
    - **Browser.js**: `npm install ali-oss` (browser build)
    - **Android/iOS**: Platform-specific SDKs

    For S3-compatible access, you can also use AWS SDKs with the OSS endpoint.
  </Accordion>

  <Accordion title="How do I handle large file uploads?">
    Use multipart upload for files larger than 100 MB:
    1. Call InitiateMultipartUpload to get an upload ID
    2. Split the file into parts and upload with UploadPart
    3. Call CompleteMultipartUpload with all part ETags

    Most SDKs provide a high-level API that handles this automatically.
  </Accordion>

  <Accordion title="How do I generate presigned URLs?">
    All SDKs support generating presigned URLs for temporary access:
    ```python
    url = bucket.sign_url('GET', 'my-object.txt', 3600)  # 1 hour
    ```
    Share the URL with users who need temporary access without credentials.
  </Accordion>

  <Accordion title="How do I handle errors and retries?">
    SDK error objects contain:
    - `status`: HTTP status code
    - `code`: OSS error code (e.g., `NoSuchKey`)
    - `message`: Error description
    - `request_id`: Unique ID for support tickets

    Implement exponential backoff for 5xx errors and 503 throttling.
  </Accordion>

  <Accordion title="Can I use the SDK from a browser?">
    Yes. Use the Browser.js SDK (`ali-oss` browser build) with STS temporary credentials. Never expose your AccessKey in client-side code.
  </Accordion>
</AccordionGroup>
