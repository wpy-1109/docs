## Use OSS-HDFS

For Hadoop workloads, enable OSS-HDFS for significant performance improvements:

- Atomic directory rename (O(1) instead of copy + delete)
- Hierarchical namespace for efficient listing
- POSIX-compatible permissions

See [OSS-HDFS](/tools/oss-hdfs/overview) for setup instructions.

## JindoSDK configuration

Optimize JindoSDK settings for your workload:

```xml
<!-- core-site.xml -->
<property>
  <name>fs.oss.committer.magic.enabled</name>
  <value>true</value>
  <description>Enable magic committer for efficient job commit</description>
</property>
<property>
  <name>fs.oss.read.buffer.size</name>
  <value>4194304</value>
  <description>Read buffer size (4 MB)</description>
</property>
<property>
  <name>fs.oss.multipart.upload.part.size</name>
  <value>67108864</value>
  <description>Multipart upload part size (64 MB)</description>
</property>
```

## Key optimizations

- **Use Jindo Committer**: Replace Hadoop S3A committer for faster job commit
- **Enable local caching**: Cache frequently accessed data blocks on local SSD
- **Tune buffer sizes**: Match read/write buffer sizes to your data patterns
- **Collocate compute**: Run compute in the same region as OSS
- **Use columnar formats**: Parquet and ORC with predicate pushdown reduce I/O
- **Partition data**: Organize data by date/category for efficient pruning
