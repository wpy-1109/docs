## Tool comparison

| Feature | ossutil | OSS Migration Service | Data Transport | ossimport | Rclone |
|---------|---------|----------------------|----------------|-----------|--------|
| **Interface** | CLI | Console | Physical device | CLI | CLI |
| **Source** | Local, OSS | S3, Azure, GCS, HTTP, OSS | On-premises | S3, Azure, local, HTTP | S3, Azure, GCS, 40+ |
| **Best for** | Up to 1 TB | 1-100 TB online | 100+ TB offline | Large distributed | Cross-cloud |
| **Resumable** | Yes | Yes | N/A | Yes | Yes |
| **Incremental** | Yes (`-u` flag) | Yes | No | Yes | Yes |
| **Bandwidth control** | Yes | Yes | N/A | Yes | Yes |
| **Parallel** | Yes | Yes | N/A | Yes | Yes |
| **Cost** | Free | Free (traffic fees apply) | Device rental fee | Free | Free |

## ossutil migration

Best for simple, direct migrations up to 1 TB:

```bash
# Upload a directory
ossutil cp -r /local/data/ oss://bucket/data/ --jobs 10 --parallel 5

# Incremental sync (only new/changed files)
ossutil cp -r /local/data/ oss://bucket/data/ -u
```

## OSS migration service

Best for large-scale cross-cloud migrations:

1. Create a migration job in the OSS console
2. Configure source (S3, Azure Blob, GCS, etc.)
3. Set destination bucket and prefix
4. Configure bandwidth limits and scheduling
5. Monitor progress in the console

## Data transport (Lightning cube)

Best for offline migration of massive datasets:

1. Order a Lightning Cube device from Alibaba Cloud
2. Copy data to the device using provided tools
3. Ship the device to the Alibaba Cloud data center
4. Data is loaded into your OSS bucket
5. Return the device
