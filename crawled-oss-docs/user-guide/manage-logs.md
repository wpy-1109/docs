# Log management

Alibaba Cloud OSS offers a comprehensive and flexible log management solution that enables enterprises to efficiently conduct action audits, troubleshoot issues, and perform data analysis. This optimizes data storage and access efficiency while enhancing operations management.

## Oss Log Management

### Real-time log query


Accessing OSS resources generates numerous access logs. The real-time log query feature integrates OSS with Simple Log Service (SLS), enabling direct queries of OSS access logs within the OSS console. This facilitates action audits, access statistics, tracing of anomalous activities, and issue localization. SLS's powerful real-time log query capabilities also allow for in-depth analysis of OSS access logs, including batch deletion of logs or hourly log metering, thus improving work efficiency and supporting data-driven decision-making.

### Log storage


Continuous access to OSS resources results in a steady stream of access logs. The log storage feature allows for the creation of log files on an hourly basis using a fixed naming convention, which are then stored in a designated bucket.

### Set log recording request header or query parameter


OSS provides a default set of access logs with fixed fields for basic analysis. For more precise tracking and analysis of user behavior or to meet specific monitoring requirements, additional fields may be necessary. OSS offers a custom log field configuration feature (user_defined_log_fields), allowing the recording of specific request headers and query parameters. This caters to a variety of custom field analysis needs and offers a more adaptable log management solution for enterprises.
