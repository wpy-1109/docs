# Methods for uploading objects to OSS

You can upload any types of files, including images, documents, and videos, to OSS buckets. Uploaded files are stored as objects in OSS buckets. An object contains a file and metadata that describes the file. You can upload a large number of objects to a bucket at a time.

## Upload methods


OSS provides the following methods to upload files:


-

[Simple upload](https://www.alibabacloud.com/help/en/oss/user-guide/simple-upload): This method is suitable for uploading a small file that is up to 5 GB in size. You can call the PutObject operation that is provided by OSS to upload a file. No configuration is required.

-

[Multipart upload](https://www.alibabacloud.com/help/en/oss/user-guide/multipart-upload): This method is suitable for uploading a large file that is up to 48.8 TB in size. You can call the InitiateMultipartUpload, UploadPart, and CompleteMultipartUpload operations to split a file into multiple parts and upload the parts in parallel. After you upload the parts, you can combine all parts into a complete object. If the upload is interrupted due to a network exception, the client manually records the failed parts and re-uploads the parts.

-

[Append upload](https://www.alibabacloud.com/help/en/oss/user-guide/append-upload-11): This method is suitable for uploading a file to append, such as a video stream. A single file to upload can be up to 5 GB in size. You can call the AppendObject operation that is provided by OSS to upload the files and generate appendable objects. You can append content to existing appendable objects. Data that is appended to an appendable object can be immediately read. Append upload supports only appendable objects.

-

[Resumable upload](https://www.alibabacloud.com/help/en/oss/user-guide/resumable-upload): This method is suitable for uploading a large file that is up to 48.8 TB in size over unstable networks. You can call the multipart upload-based method that is provided by OSS SDKs, such as the `uploadFile` method provided by OSS SDK for Java, to automatically record the upload progress on the client and resume the upload from the position at which the upload stops.

-

[Form upload](https://www.alibabacloud.com/help/en/oss/user-guide/form-upload): This method is suitable for uploading an object that is up to 5 GB in size in an HTML page by initiating an HTTP POST request to OSS. You can use the policy generated by the server to limit the attributes of an object that you want to upload, such as the object size and type.

-

[Object uploads by using presigned URLs](https://www.alibabacloud.com/help/en/oss/user-guide/upload-files-using-presigned-urls): This method is suitable for granting a third-party user permissions to upload objects less than 5 GB in size. Owner of objects can generate presigned URLs and specify their validity periods to allow third-party users to upload objects with no need for AccessKey pairs. The presigned URLs become invalid automatically after expiration.

## References


-

You can use the upload progress bar that is provided by OSS SDKs to display the upload progress and the remaining time that is required to complete the upload. For more information, see [Upload progress bar](https://www.alibabacloud.com/help/en/oss/user-guide/upload-progress-bar).

-

After you upload an object to OSS, you can send a callback request to a specified application server by using upload callbacks. For more information, see [Upload callbacks](https://www.alibabacloud.com/help/en/oss/user-guide/upload-callbacks-12#concept-ywd-dlb-5db).

-

If you want to control the caching, downloading, and processing of uploaded objects, you can configure object metadata, such as standard HTTP headers including Content-Type, when you upload objects. For more information, see [Manage object metadata](https://www.alibabacloud.com/help/en/oss/user-guide/manage-object-metadata-10/#concept-lkf-swy-5db).

-

We recommend that you upload files directly from your client to OSS. This solution accelerates uploads and reduces the resource usage of the application server by eliminating the need to transfer objects to and from the application server. For more information, see [Overview](https://www.alibabacloud.com/help/en/oss/user-guide/uploading-objects-to-oss-directly-from-clients/).

-

Refer to [IMG implementation modes](https://www.alibabacloud.com/help/en/oss/user-guide/img-implementation-modes#concept-m4f-dcn-vdb) if you want to perform operations such as watermarking images, converting image formats, and querying information about images.

-

Refer to [Media processing](https://www.alibabacloud.com/help/en/oss/user-guide/introduction-2/) if you want to transcode or capture frames from uploaded videos.

-

Refer to [Weboffice online preview](https://www.alibabacloud.com/help/en/oss/user-guide/online-object-preview) and [Online document editing with WebOffice](https://www.alibabacloud.com/help/en/oss/user-guide/online-object-editing) if you want to preview or edit Powerpoint or Word documents online.

-

You can add signature information to the URL of an uploaded object and share the signed URL to allow third parties to access the object. For more information, see [Download objects using presigned URLs](https://www.alibabacloud.com/help/en/oss/user-guide/how-to-obtain-the-url-of-a-single-object-or-the-urls-of-multiple-objects).
