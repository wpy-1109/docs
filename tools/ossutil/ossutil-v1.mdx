<Warning>
ossutil 1.0 is a legacy version. Alibaba Cloud recommends upgrading to [ossutil 2.0](/tools/ossutil/overview) for new features, improved performance, and better security.
</Warning>

## Key improvements in ossutil 2.0

- Multi-level command syntax (API-level and high-level commands)
- Simplified configuration with multiple profiles via `--profile`
- Path, file size, modified time, and metadata filters
- Flexible output formats: JSON, YAML, XML via `--output-format`
- Environment variable support for credentials
- `--dry-run` option for safe testing

## ossutil 1.0 common commands

| Command | Description |
|---------|-------------|
| `config` | Create configuration file |
| `cp` | Upload, download, or copy objects |
| `ls` | List buckets, objects, or multipart uploads |
| `mb` | Create a bucket |
| `rb` | Delete a bucket |
| `rm` | Delete objects |
| `stat` | View bucket/object details |
| `cat` | Output object content to stdout |
| `du` | Query storage usage |
| `sign` | Generate presigned URL |
| `restore` | Restore archived objects |
| `create-symlink` | Create symbolic links |
| `read-symlink` | Read symbolic link target |
| `hash` | Calculate CRC-64 or MD5 hash |
| `probe` | Monitor and troubleshoot access |
| `mkdir` | Create a directory |

## Migration guide

To migrate from ossutil 1.0 to 2.0:

1. Install ossutil 2.0 alongside 1.0
2. Run `ossutil config` to set up credentials
3. Update scripts to use the new command syntax
4. Most commands (`cp`, `ls`, `rm`, `mb`, `rb`) have the same syntax in 2.0
5. Test with `--dry-run` before running in production
