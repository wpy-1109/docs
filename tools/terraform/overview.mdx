The Alibaba Cloud Terraform provider (`alicloud`) supports managing OSS buckets, objects, and configurations as infrastructure as code.

## Setup

<Steps>
  <Step title="Install Terraform">
    Download and install [Terraform](https://developer.hashicorp.com/terraform/install) for your platform.
  </Step>
  <Step title="Configure the provider">
    Create a `main.tf` file with the Alibaba Cloud provider:

    ```hcl
    terraform {
      required_providers {
        alicloud = {
          source  = "aliyun/alicloud"
          version = ">= 1.200.0"
        }
      }
    }

    provider "alicloud" {
      access_key = var.access_key
      secret_key = var.secret_key
      region     = var.region
    }

    variable "access_key" {
      type      = string
      sensitive = true
    }

    variable "secret_key" {
      type      = string
      sensitive = true
    }

    variable "region" {
      type    = string
      default = "cn-hangzhou"
    }
    ```
  </Step>
  <Step title="Initialize Terraform">
    ```bash
    terraform init
    ```
  </Step>
</Steps>

## OSS resources

| Resource | Description |
|----------|-------------|
| `alicloud_oss_bucket` | Create and manage buckets (ACL, versioning, lifecycle, encryption, CORS, website) |
| `alicloud_oss_bucket_object` | Upload and manage objects |
| `alicloud_oss_bucket_acl` | Manage bucket ACLs |
| `alicloud_oss_bucket_policy` | Configure bucket policies |
| `alicloud_oss_bucket_versioning` | Manage versioning settings |
| `alicloud_oss_bucket_lifecycle` | Manage lifecycle rules |

### Data sources

| Data Source | Description |
|------------|-------------|
| `alicloud_oss_buckets` | List existing buckets (supports name filtering) |
| `alicloud_oss_bucket_objects` | List objects within a bucket |
