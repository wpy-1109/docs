ossbrowser is the official graphical management tool for Alibaba Cloud OSS. It provides a visual, desktop-based way to manage buckets and objects with drag-and-drop uploads, file previews, and access control -- similar to a file explorer.

<Warning>
When uploading or downloading files larger than 10 GB over the public internet, network instability may cause transfer failures. For large file transfers, use [ossutil](/tools/ossutil/overview) (CLI) instead.
</Warning>

## System requirements

| Platform | Architecture | Supported |
|----------|-------------|-----------|
| Windows | x86 (32-bit), x86-64 (64-bit) | Yes |
| macOS | x86-64 | Yes |
| Linux | x86-64 (requires GUI desktop) | Yes |

## Installation

<Tabs>
  <Tab title="Windows">
    <Steps>
      <Step title="Download the installer">
        Download the package for your system architecture:

        | Architecture | Download |
        |-------------|----------|
        | Windows x64 | [oss-browser-win32-x64.zip](https://gosspublic.alicdn.com/ossbrowser/1.18.0/oss-browser-win32-x64.zip) |
        | Windows x32 | [oss-browser-win32-ia32.zip](https://gosspublic.alicdn.com/ossbrowser/1.18.0/oss-browser-win32-ia32.zip) |
      </Step>
      <Step title="Extract the archive">
        Unzip the downloaded file to a folder of your choice.
      </Step>
      <Step title="Launch ossbrowser">
        Open the extracted folder and double-click `oss-browser.exe`.
      </Step>
    </Steps>
  </Tab>
  <Tab title="macOS">
    <Steps>
      <Step title="Download the package">
        Download [oss-browser-darwin-x64.zip](https://gosspublic.alicdn.com/ossbrowser/1.18.0/oss-browser-darwin-x64.zip).
      </Step>
      <Step title="Extract the archive">
        Unzip the downloaded file.
      </Step>
      <Step title="Launch ossbrowser">
        In the `oss-browser-darwin-x64` folder, right-click `oss-browser.app` and select **Open**.
      </Step>
      <Step title="Approve Gatekeeper">
        macOS may show a security dialog. Click **Open** to allow the app, then click **Allow** if prompted.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Linux">
    <Steps>
      <Step title="Install a GUI desktop">
        ossbrowser requires a graphical desktop environment. If your Linux server does not have one, install a desktop environment first (e.g., GNOME or XFCE).
      </Step>
      <Step title="Install dependencies">
        Install the required shared libraries:

        ```bash
        # Ubuntu / Debian
        sudo apt update
        sudo apt install libx11-xcb1 libxtst6 libxss1 libgconf-2-4 libgtk2.0-0

        # CentOS / RHEL
        sudo yum install libXtst libXScrnSaver gtk2 GConf2
        ```
      </Step>
      <Step title="Download and extract">
        ```bash
        wget https://gosspublic.alicdn.com/ossbrowser/1.18.0/oss-browser-linux-x64.zip
        unzip oss-browser-linux-x64.zip
        ```
      </Step>
      <Step title="Launch ossbrowser">
        ```bash
        cd oss-browser-linux-x64
        ./oss-browser
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

For all versions and release notes, see the [ossbrowser GitHub releases](https://github.com/aliyun/ossbrowser/releases).

<Tip>
The current version is **1.18.0**. ossbrowser is best suited for ad-hoc management tasks with small-to-medium files. For scripting, automation, or files larger than 10 GB, use [ossutil](/tools/ossutil/overview) instead.
</Tip>

## Login methods

<Tabs>
  <Tab title="AccessKey login">
    Use your AccessKey ID and AccessKey Secret for full account access.

    1. Select **AK Login** mode
    2. Enter your AccessKey ID and Secret
    3. Optionally set a preset OSS path to restrict scope
    4. Select the region/endpoint
    5. Click **Login**
  </Tab>
  <Tab title="STS token login">
    Use temporary security credentials for scoped, time-limited access. Ideal for granting limited access to sub-users.

    1. Select **STS Token Login** mode
    2. Enter the temporary AccessKey ID, Secret, and Security Token
    3. Set the preset OSS path
    4. Click **Login**
  </Tab>
</Tabs>

## Key features

| Feature | Description |
|---------|-------------|
| Bucket management | Create, delete, and configure buckets |
| File upload/download | Drag-and-drop, batch operations, resumable transfers |
| File preview | Preview images, text files, and other formats in-app |
| Access control | Set ACLs at bucket and object level |
| Cross-region support | Access buckets across Alibaba Cloud regions |
| Search and filter | Locate files within buckets |
| Directory-like view | Browse objects in a folder-style hierarchy |
| Operation logs | View detailed logs of operations performed |

## Common operations

### Upload files

1. Navigate to the target bucket and directory
2. Click **Upload** or drag-and-drop files into the window
3. Monitor upload progress in the transfer panel
4. Large files automatically use multipart upload with resumable support

### Download files

1. Select one or more files/directories
2. Click **Download** and choose the local destination
3. Downloads support resume if interrupted

### Manage permissions

1. Right-click a bucket or object
2. Select **Set ACL** from the context menu
3. Choose from `private`, `public-read`, or `public-read-write`
