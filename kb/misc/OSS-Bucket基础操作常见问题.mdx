# OSS Bucket基础操作常见问题

## Bucket名称是否全局唯一？

**是的。** OSS Bucket名称在所有地域（包括中国大陆和海外地域）必须全局唯一。即使在不同地域，也不能创建同名Bucket。

- 如果创建Bucket时提示名称已存在，说明该名称已被其他用户占用（包括其他阿里云账号）。
- 没有提前查询Bucket名称是否可用的独立API，创建时系统会自动校验并返回结果。
- Bucket创建超时后，需要先确认是否实际创建成功。已创建成功的Bucket不能重复创建同名Bucket。

**建议**：Bucket名称建议使用有业务含义的前缀（如公司缩写），避免使用过于通用的名称。

## 如何查看Bucket中的文件数量？

在OSS控制台中，进入目标Bucket，在**概览**页面即可看到当前Bucket的文件总数和存储量等基本信息。

如果需要查看特定前缀（目录）下的文件数量，可使用ossutil的`du`命令：

```bash
ossutil du oss://bucket-name/prefix/
```

## OSS有容量限制吗？

**没有。** 单个Bucket和整个OSS服务均没有存储容量上限，您可以持续上传数据。控制台显示的容量是当前已使用的存储量，不是容量上限。

如果您购买了存储包（资源包），存储包有容量规格，超出部分按量计费。存储包的容量不是OSS的使用上限，只是预付费抵扣额度。

## AccessKey Secret丢失了怎么办？

AccessKey Secret只在创建时显示一次，之后无法再次查看。如果丢失，只能重新创建一组新的AccessKey。

操作步骤：
1. 登录[RAM控制台](https://ram.console.aliyun.com)。
2. 在**身份管理** > **用户**中找到对应RAM用户。
3. 点击用户名，进入**认证管理**标签页。
4. 创建新的AccessKey并妥善保存。
5. 更新使用旧AccessKey的所有应用配置。
6. 确认业务正常后，禁用并删除旧的AccessKey。

## 如何查看历史存储用量？

在OSS控制台的**数据统计** > **基础数据**中可以查看存储用量趋势，但**仅支持查询最近180天**的数据。超过180天的历史数据无法通过控制台查看。

如果需要长期保留存储用量数据，建议：
- 通过[云监控](https://cloudmonitor.console.aliyun.com)配置自定义监控，将存储指标写入日志长期保存。
- 使用费用中心的账单数据间接了解各月存储费用。

## 如何查看API调用次数？

在OSS控制台的**数据统计** > **基础数据**中可以查看请求次数统计，包括Get类请求和Put类请求的次数趋势。

也可以通过云监控查看更详细的API调用监控数据：
1. 登录云监控控制台。
2. 选择**云服务监控** > **对象存储OSS**。
3. 选择目标Bucket查看各类请求的监控图表。

## 能否按文件类型统计存储量（如图片、视频分别多少）？

**OSS控制台不支持按文件类型分类统计。** 用量查询仅提供Bucket级别的总存储量。

如需按文件类型统计，可通过以下方式实现：
- 使用[数据索引](https://help.aliyun.com/document_detail/2743800.html)功能对文件建立索引，支持按文件后缀等条件聚合查询。
- 通过ossutil结合脚本按后缀分类统计。
