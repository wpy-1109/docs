# OSS有效请求率报警排查与处理

## 症状

用户收到云监控发送的OSS报警通知，典型表现包括：

- 收到"(Bucket)有效请求率(Warn)"报警，提示"连续5个周期监控值"低于阈值（如50%）。
- 报警级别显示为CRITICAL，但查看告警详情时最终状态为Normal。
- 频繁收到有效请求率报警短信或邮件通知，但业务运行正常。

## 原因分析

### 有效请求率下降的常见原因

有效请求率 = 有效请求数 / 总请求数 × 100%。当大量请求返回4xx或5xx状态码时，有效请求率会下降。

1. **客户端错误（4xx）导致有效请求率降低**：
   - 权限配置错误导致大量403 Forbidden请求。
   - 请求的Object不存在，产生大量404 Not Found请求。
   - 签名过期或时间偏差超过15分钟，导致403请求。

2. **爬虫或恶意扫描**：外部爬虫扫描不存在的路径，产生大量404请求，拉低有效请求率。

3. **程序逻辑缺陷**：应用代码中存在重试逻辑错误或路径拼接错误，导致重复发起无效请求。

### 告警状态为Normal但级别为CRITICAL

这是云监控的正常机制。当某个监控周期检测到有效请求率低于阈值时触发CRITICAL告警，后续周期恢复正常则状态自动变为Normal。告警记录保留了历史触发时的级别信息。

## 解决方案

### 步骤一：查看报警详情

1. 登录[云监控控制台](https://cloudmonitor.console.aliyun.com)。
2. 在左侧导航栏选择**报警服务** > **报警历史**。
3. 根据时间范围和产品类型（对象存储OSS）筛选报警记录。
4. 查看报警触发时间、持续周期数和恢复时间。

### 步骤二：分析请求日志

1. 在OSS控制台对应Bucket的**日志管理**中启用日志查询（需提前开通日志服务SLS）。
2. 在SLS控制台查询报警时段的请求日志，重点关注4xx和5xx状态码的请求。
3. 分析错误请求的来源IP、请求路径和错误码，确定根因。

### 步骤三：针对性修复

| 根因 | 处理方法 |
|------|---------|
| 大量404请求 | 检查客户端请求的Object路径是否正确；排查是否有爬虫扫描，可通过Bucket Policy限制来源IP |
| 大量403请求 | 检查签名逻辑和AccessKey是否正确；确认RAM权限策略是否完整覆盖所需操作 |
| 外部爬虫 | 配置Bucket Policy或Referer防盗链，限制非法来源访问 |
| 程序重试风暴 | 优化客户端重试策略，增加退避间隔（Exponential Backoff） |

### 步骤四：调整报警规则

如果业务场景中少量4xx请求属于正常现象（如用户访问不存在的资源），可以适当调整报警阈值：

1. 在云监控控制台找到对应的报警规则。
2. 将有效请求率阈值从默认值调整为适合业务的比例（如调低至30%）。
3. 增加连续触发周期数，避免瞬时波动触发误报。

## 预防措施

- 开通OSS访问日志并定期审查，及时发现异常请求模式。
- 为公开访问的Bucket配置Referer白名单和IP黑名单。
- 客户端实现合理的重试策略，避免无效请求堆积。
- 根据业务实际情况设置合理的监控阈值，减少误报。
