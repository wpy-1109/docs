### Q1：OSS未开启内容安全服务，平台会主动检测和封禁违规图片吗？

**回答：** 会。即使您未开启 OSS 内容安全检测服务，阿里云平台仍会根据法律法规和服务条款，对存储在 OSS 中的内容进行基础合规扫描。如果检测到违法违规内容（如涉黄、涉政、暴恐类），平台会冻结违规文件并发送通知。

内容安全检测服务是可选的增值服务，开启后您可以主动对指定 Bucket 进行更细粒度的检测，并根据检测结果自动或手动处理。

### Q2：收到违规文件通知后该怎么处理？

**回答：**

1. 根据通知中的违规文件路径，在 OSS 控制台的文件列表中找到对应文件。
2. 确认文件内容是否违规。
3. 删除违规文件。

```
# 在文件列表中输入完整文件路径搜索
app/2026-03-19/ae29837438ff4f7fa7a72eb4878b777a.jpeg
```

### Q3：内容安全检测到违规图片后，能自动删除吗？

**回答：** 不能。OSS 内容安全检测服务仅提供检测结果（风险标签和置信分），**不会自动删除**文件。您需要根据检测结果，通过以下方式处理：

- 通过回调通知获取违规信息，编写自动化脚本调用 DeleteObject API 删除
- 在控制台手动删除
- 使用 ossutil 批量删除

### Q4：我觉得是误判，如何申诉？

**回答：** 您可以在 OSS 控制台或通过工单提交申诉。建议提供以下信息：
- 违规文件的完整路径
- 违规通知中的检测标签
- 您认为误判的理由

### Q5：如何预防用户上传违规内容？

**回答：** OSS 本身不提供上传时的实时内容拦截功能。建议在业务层实现以下方案：

1. **上传后异步检测**：开通内容安全服务，对新上传文件进行检测，发现违规及时处理。
2. **业务侧前置过滤**：在应用层增加内容审核逻辑，上传前进行预检测。
3. **设置事件通知**：配置 OSS 事件通知，在文件上传时触发函数计算进行审核。
