### 症状

删除 Bucket 中的 `oss-accesslog/` 目录后，该目录会自动重新生成，无法彻底删除。

### 原因

`oss-accesslog/` 是 OSS **日志转存**功能自动生成的目录。当 Bucket 开启了日志转存后，系统会持续向该目录写入新的访问日志文件。即使手动删除该目录及其下的文件，只要日志转存功能仍处于开启状态，系统就会继续生成新的日志文件。

### 解决方案

**步骤一：关闭日志转存功能**

1. 登录 OSS 管理控制台。
2. 进入目标 Bucket。
3. 在左侧导航栏选择 **日志管理** > **日志转存**。
4. 关闭日志转存功能。

**步骤二：删除已有日志文件**

关闭日志转存后，`oss-accesslog/` 目录将不再自动生成新文件。此时可以手动或通过 ossutil 批量删除该目录下的所有文件。

```bash
ossutil rm oss://your-bucket/oss-accesslog/ -r -f
```

### 补充说明

如果需要保留日志记录用于审计或分析，建议使用 **实时日志查询**（集成日志服务 SLS），无需占用 Bucket 存储空间，且支持更灵活的查询和分析能力。
