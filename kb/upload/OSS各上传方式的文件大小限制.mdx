### 各上传方式文件大小限制速查表

| 上传方式 | 单文件大小限制 | 说明 |
|----------|----------------|------|
| **控制台上传** | 5 GB | Web 控制台直接上传 |
| **PostObject** | 5 GB | 表单上传，常用于 Web 端直传 |
| **PutObject** | 5 GB | 简单上传，适用于小文件 |
| **分片上传** | 48.8 TB | 每个分片最小 100 KB（最后一个除外），最多 10000 个分片 |
| **ossbrowser** | 48.8 TB（理论值） | 建议单次不超过 1 GB，大文件建议使用 ossutil |
| **ossutil** | 48.8 TB | 自动使用分片上传，推荐用于大文件 |
| **SDK** | 48.8 TB | 使用分片上传 API 时 |

### 常见问题

**Q：通过控制台上传超过 5GB 的文件怎么办？**

使用 ossutil 命令行工具或 SDK 的分片上传功能。ossutil 会自动对大文件进行分片上传：

```bash
ossutil cp /local/large-file.tar.gz oss://your-bucket/path/ 
```

**Q：ossbrowser 上传大文件卡住或失败怎么办？**

ossbrowser 是图形化工具，在处理大文件时可能出现内存不足或连接超时。建议：
- 单次上传文件不超过 1 GB
- 大文件使用 ossutil 命令行工具上传
- 多个目录分批上传

**Q：SDK 分片上传需要前端自己切割文件吗？**

不需要。使用阿里云 OSS SDK 的分片上传功能时，SDK 会自动将文件按照每个分片进行切割和上传，无需开发者额外处理。
