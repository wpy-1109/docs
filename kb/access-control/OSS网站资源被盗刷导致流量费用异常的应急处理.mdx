# OSS 网站资源被盗刷导致流量费用异常的应急处理

OSS 外网流出流量费用突然暴增，远超正常业务水平，怀疑资源被盗刷或恶意下载。本文介绍紧急止损和安全加固的操作步骤。

## 常见症状

- 收到阿里云欠费短信，金额异常增高（几百甚至上千元/天）。
- OSS 控制台的流量监控显示外网流出流量远超日常水平。
- 费用中心账单中"外网流出流量"项目金额异常。
- Bucket 为公共读权限，且未配置任何访问限制。

## 原因

1. **Bucket 为公共读权限**：任何人都可以通过 URL 直接下载文件，攻击者利用此特性大量拉取资源。
2. **资源 URL 被第三方网站引用**：视频、图片等被其他网站直接嵌入引用（盗链）。
3. **AccessKey 泄露**：AK 被恶意使用，大量下载 Bucket 中的文件。
4. **爬虫/自动化工具大量访问**：搜索引擎爬虫或恶意爬虫频繁访问 OSS 资源。

## 紧急止损（立即执行）

### 第一步：将 Bucket 设为私有

1. 登录 OSS 控制台 > 目标 Bucket > **权限控制** > **读写权限**。
2. 将 ACL 改为**私有（private）**。
3. 开启**阻止公共访问**。

> 注意：这会导致所有通过裸 URL 访问的链接立即失效。如果有正常业务依赖公共读，需要先准备替代方案（如 CDN 回源）。

### 第二步：配置防盗链

1. 进入 **权限控制** > **防盗链**。
2. 设置 Referer 白名单，只允许你自己网站的域名。
3. 禁止空 Referer（防止直接通过 URL 访问）。

### 第三步：如怀疑 AK 泄露

1. 立即在 RAM 控制台禁用可疑的 AccessKey。
2. 创建新 AK，更新应用配置。

### 第四步：设置费用告警

1. 在费用中心配置**可用额度预警**或**每日消费预警**。
2. 设置合理的阈值，便于第一时间发现异常。

## 安全加固（长期方案）

1. **使用 CDN + 私有 Bucket**：通过 CDN 对外提供访问，OSS Bucket 保持私有。CDN 侧可配置更丰富的安全策略（IP 黑名单、频次限制等），且 CDN 流量费低于 OSS 外网流量费。
2. **配置 Bucket Policy IP 限制**：只允许已知的公网 IP 和 VPC 访问。
3. **使用签名 URL**：所有文件通过签名 URL 访问，签名 URL 有有效期。
4. **开启日志审计**：在 Bucket 的日志管理中开启实时日志查询，便于快速排查异常访问来源。

## 排查异常来源

1. 进入 OSS 控制台 > 目标 Bucket > **日志管理** > **实时查询**。
2. 使用以下查询条件：
   - 按 `operation:GetObject` 筛选下载请求。
   - 按 `client_ip` 统计来源 IP 分布。
   - 按 `referer` 查看引用来源。
   - 按 `access_id` 查看使用的 AK。

## 常见问题

**Q: 被盗刷的流量费用能退吗？**

因 Bucket 配置不当（如公共读）导致的流量盗刷，属于用户侧安全管理责任，阿里云通常无法退款。建议尽快止损并加固配置。

**Q: 为什么配置了 CDN 还有 OSS 外网流量？**

可能原因：
1. 应用中仍有部分 URL 使用了 OSS 默认域名。
2. Bucket 为公共读，有人直接通过 OSS 域名访问（绕过了 CDN）。
3. CDN 回源请求的流量也会计入 OSS 回源流量（但费用通常由 CDN 回源流量包抵扣）。
