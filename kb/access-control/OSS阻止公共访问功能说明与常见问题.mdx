# OSS 阻止公共访问功能说明与常见问题

修改 Bucket ACL 为公共读时报错 `AccessDenied`（EC 码 `0015-00000501`），或已设置公共读但文件仍无法匿名访问。本文解释"阻止公共访问"功能的作用及处理方法。

## 常见症状

**症状一**：修改 Bucket ACL 为公共读或公共读写时，报错：

```
AccessDenied
ecCode: 0015-00000501
```

**症状二**：Bucket ACL 已设为公共读，但通过 URL 直接访问文件仍返回 `AccessDenied`。

**症状三**：在 Bucket Policy 中设置了允许匿名访问的策略，但匿名请求仍被拒绝。

## 原因

OSS 的"阻止公共访问"功能开启后，会覆盖 ACL 和 Bucket Policy 中的公共访问权限设置：

- **不允许**将 Bucket ACL 或 Object ACL 设置为 `public-read` 或 `public-read-write`。
- **不允许** Bucket Policy 中对匿名用户（`Principal: *`）的 Allow 策略生效。
- 即使之前已设置了公共读 ACL，开启该功能后公共访问也会被阻断。

该功能分为两个级别：
- **账号级别**：在 OSS 概览页设置，影响账号下所有 Bucket。
- **Bucket 级别**：在具体 Bucket 的权限控制中设置，只影响该 Bucket。

## 解决方案

### 如果需要公共访问

1. 进入 OSS 控制台 > 目标 Bucket > **权限控制** > **阻止公共访问**。
2. 关闭 Bucket 级别的阻止公共访问。
3. 同时检查账号级别设置：OSS 控制台 > **概览** > **安全配置** > **阻止公共访问**，确认未在账号层面开启。
4. 关闭后再修改 Bucket ACL 为公共读。

### 如果不需要公共访问（推荐）

保持阻止公共访问开启，使用以下安全访问方式代替：

- **预签名 URL**：为每个文件生成带有效期的签名链接。
- **CDN + 私有 Bucket 回源**：通过 CDN 对外提供访问，OSS Bucket 保持私有。
- **STS 临时凭证**：客户端通过临时凭证访问。

### 评估影响

将 Bucket 从公共读改为私有并开启阻止公共访问前，评估以下影响：

1. **网站或 App 中直接引用的 OSS 文件 URL**：会全部失效，需改为签名 URL。
2. **第三方系统通过裸 URL 拉取资源**：会被拒绝，需配置认证信息。
3. **CDN 回源**：如已配置 CDN，需在 CDN 侧开启"私有 Bucket 回源"功能。

## 预防措施

- 生产环境建议保持阻止公共访问开启，使用签名 URL 或 CDN 回源方式提供访问。
- 阿里云健康体检中"Bucket 开放公共读写权限"为高危项，建议根据实际业务需求评估并整改。

## 仍有问题？

如果关闭了阻止公共访问后仍无法修改 ACL，检查是否在账号级别也开启了该功能。如果账号级别也已关闭但仍报错，请提供 RequestId 提交工单。
