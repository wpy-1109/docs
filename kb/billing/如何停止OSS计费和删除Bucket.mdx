# 如何停止OSS计费和删除Bucket

不再使用 OSS 服务后，需要手动删除 Bucket 中的数据和 Bucket 本身，才能停止计费。OSS 没有"一键关闭"按钮。

## 适用场景

- 不再需要 OSS 服务，想停止所有费用
- 收到欠费通知，想彻底停用 OSS
- 想删除某个不用的 Bucket

## 前提条件

- 账户状态正常（未欠费）。欠费期间无法在控制台操作，需先充值恢复
- 已备份需要保留的数据（删除后不可恢复）

## 操作步骤

### 1. 备份重要数据

删除操作不可逆，请先确认 Bucket 中的数据是否需要备份。

### 2. 删除 Bucket 中的所有文件

**通过控制台删除：**
1. 登录 [OSS 控制台](https://oss.console.aliyun.com/)
2. 点击目标 Bucket
3. 进入"文件管理"，选择并删除所有文件

**通过命令行工具 ossutil 删除：**
```bash
ossutil rm oss://bucket-name -r -f
```

> **注意**：如果开启了版本控制，还需要删除历史版本文件。否则历史版本仍会产生存储费用。

### 3. 清理关联配置

删除 Bucket 前，需要先清理以下配置（如有）：
- 关闭跨区域复制任务
- 删除绑定的自定义域名
- 关闭数据索引功能
- 清理未完成的分片上传碎片

### 4. 删除 Bucket

**通过控制台删除：**
1. 在 Bucket 列表中找到目标 Bucket
2. 点击右侧"..."，选择"删除 Bucket"
3. 输入 Bucket 名称确认删除

**通过命令行删除：**
```bash
ossutil rm oss://bucket-name -b
```

### 5. 确认不再产生费用

删除所有 Bucket 后，检查：
- [费用中心](https://billing-cost.console.aliyun.com/) 确认无新增 OSS 账单
- 账单有 3-4 小时延迟，删除当天可能还会看到少量费用

## 重要提醒

**OSS 文件删除后无法恢复**

一旦删除文件或 Bucket，数据将永久丢失。除非开启了版本控制且未删除历史版本，否则无法找回。

**关于同城冗余**

同城冗余是 Bucket 级别的配置，创建后无法关闭或更改。如果想从同城冗余切换到本地冗余，需要创建新的 Bucket 并迁移数据。

**不使用但不想删除**

如果只是暂时不用，可以保留 Bucket 但清空其中的文件，这样只会产生极少的费用（空 Bucket 不收存储费）。

## 常见问题

**Q：删除文件后控制台还显示有容量怎么办？**
控制台显示有 1-3 小时延迟，删除后等待几小时再查看即可。

**Q：欠费了无法操作怎么办？**
需要先充值结清欠费，等服务恢复后再操作删除。或者等 15 天后数据会被自动清理（但这期间仍会产生费用）。

**Q：只删文件不删 Bucket 还会收费吗？**
空 Bucket 不收存储费。但如果有 API 请求仍会产生少量请求费用。建议不用的 Bucket 直接删除。
