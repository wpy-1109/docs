# OSS不用了还在扣费怎么办

服务器停了、业务不做了，但 OSS 还在扣费？这是因为 OSS 是独立于 ECS 的存储服务，不会随 ECS 停机或到期而自动停止。

## 常见症状

- ECS 已到期/停机，但 OSS 仍在扣费
- 账号一直在产生 OSS 费用，但自己并不知道开通过 OSS
- 每天/每月有小额扣费，不知道来源

## 原因

### 1. Bucket 中仍有数据

OSS 只要存储了数据，就会持续产生存储费用（按小时计费）。即使没有人访问，存储费用仍然存在。

### 2. 有关联服务在使用 OSS

某些阿里云服务会自动在 OSS 中创建 Bucket 或存储数据：
- **ECS 快照**：快照底层存储在 OSS 中
- **日志服务**：日志投递到 OSS
- **视频点播**：视频文件存储在 OSS
- **其他服务**：部分服务会自动创建 OSS Bucket

### 3. 开启了版本控制

开启版本控制后，删除文件只是添加删除标记，历史版本数据仍然保留并计费。

### 4. 有分片上传碎片

未完成的分片上传会在 Bucket 中留下碎片文件，占用存储并产生费用。

## 解决方案

### 步骤一：确认费用来源

1. 登录 [费用中心 > 账单详情](https://billing-cost.console.aliyun.com/finance/expense-report/expense-detail-by-instance)
2. 选择"对象存储 OSS"
3. 查看具体哪个计费项在产生费用

### 步骤二：检查所有 Bucket

1. 登录 [OSS 控制台](https://oss.console.aliyun.com/)
2. 查看所有 Bucket 及其存储用量
3. 特别注意各地域是否有遗忘的 Bucket

### 步骤三：删除不需要的数据和 Bucket

对于确认不再需要的 Bucket：

1. 备份需要保留的数据
2. 清空 Bucket 中的所有文件（包括历史版本和碎片）
3. 删除 Bucket

详细步骤参见 [如何停止 OSS 计费和删除 Bucket](./如何停止OSS计费和删除Bucket.md)

### 步骤四：检查关联服务

如果删除了所有 Bucket 仍有费用：
- 检查 ECS 快照是否还在（快照费用在 ECS 账单下但底层走 OSS）
- 检查日志服务、视频点播等服务是否在写入 OSS
- 检查是否有数据索引等增值功能仍在运行

## 常见问题

**Q：OSS 文件删除后是不是就不扣钱了？**
文件删除后不再产生存储费用。但如果 Bucket 仍然存在且有 API 请求，会产生少量请求费用。建议不用的 Bucket 直接删除。

**Q：把 Bucket 里的文件都删了，Bucket 空着会扣费吗？**
空 Bucket 不收存储费用，但保持开着状态可能会有极少量请求费用。建议直接删除 Bucket。

**Q：不知道 OSS 里存的是什么，能删吗？**
建议先登录 OSS 控制台查看 Bucket 内的文件内容，确认是否为重要数据。如果不确定，可以先下载备份后再删除。
