# OSS 防盗链（Referer）配置导致图片无法显示或访问被拒绝

## 症状

- OSS 中的图片在网站上正常显示，但在某些场景下无法加载
- 本地 HTML 文件中引用 OSS 图片无法显示
- 浏览器直接打开 OSS 图片链接返回 403 错误
- 图片"第一次能打开，第二次就打不开了"
- 错误提示中包含 `You are denied by bucket referer policy`

## 原因

Bucket 配置了防盗链（Referer 白名单/黑名单），但访问请求的 Referer 头不满足策略要求：

1. **不允许空 Referer**：浏览器地址栏直接打开、本地 HTML 文件访问、部分 App 内嵌浏览器不发送 Referer，被策略拒绝
2. **白名单未覆盖来源域名**：网站域名未加入白名单
3. **与 CORS 冲突**：防盗链返回 403 时不带 CORS 头，浏览器可能将其报告为跨域错误而非 403

## 解决方案

### 方案 1：调整防盗链白名单

1. 登录 [OSS 控制台](https://oss.console.aliyun.com/)
2. 进入目标 Bucket >「数据安全」>「防盗链」
3. 在白名单中添加需要访问的来源域名

**白名单格式示例**：
```
*.yourdomain.com
yourdomain.com
*.anotherdomain.com
```

### 方案 2：允许空 Referer

如果需要支持以下场景，必须勾选「允许空 Referer」：
- 浏览器地址栏直接访问 URL
- 本地 HTML 文件中引用 OSS 图片
- 部分移动端 App 内嵌浏览器
- API 或后端服务直接请求（不带 Referer）
- CORS 预检请求（OPTIONS 请求）

### 方案 3：检查防盗链与 CORS 的配合

如果同时使用防盗链和 CORS，需确保：
- 防盗链白名单中包含 CORS 来源域名
- 建议允许空 Referer，否则 OPTIONS 预检请求会被拒绝

### 方案 4：排查"第一次能打开第二次打不开"

这种问题通常不是防盗链导致的，而是浏览器缓存 + Vary: Origin 未勾选导致的 CORS 问题。参考 CORS 配置文章解决。

## 防盗链配置建议

| 场景 | 推荐配置 |
|------|---------|
| 仅限自己的网站访问 | 白名单 + 允许空 Referer |
| 防止外站盗用 | 白名单 + 允许空 Referer（否则直链访问也会被拦截） |
| 完全禁止外部访问 | 白名单 + 不允许空 Referer + 配合签名 URL |
