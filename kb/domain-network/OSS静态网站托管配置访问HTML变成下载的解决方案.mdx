# OSS 静态网站托管配置：访问 HTML 变成下载的解决方案

## 症状

将 HTML 文件上传到 OSS 后，通过浏览器访问时文件被下载而不是在浏览器中渲染显示。具体表现：
- 访问 `https://bucket.oss-cn-xxx.aliyuncs.com/index.html` 时浏览器弹出下载对话框
- 网站页面无法正常展示，所有 HTML 文件都变成下载
- 配置了静态网站托管但通过 Bucket 域名访问仍然下载

## 原因

OSS 默认域名（`bucket-name.oss-cn-xxx.aliyuncs.com`）强制以下载方式处理所有请求，这是 OSS 的安全策略，不能通过配置改变。

## 解决方案

### 方案 1：绑定自定义域名（推荐）

这是唯一正确的解决方案。通过自定义域名访问 OSS 时，HTML 文件会在浏览器中正常渲染。

1. 准备一个已完成 ICP 备案的域名（中国内地地域 Bucket 要求）
2. 在 OSS 控制台绑定自定义域名（参考「OSS 绑定自定义域名常见问题与配置指南」）
3. 在 DNS 服务商处添加 CNAME 记录
4. 通过自定义域名访问网站

### 方案 2：配置静态网站托管

绑定自定义域名后，进一步配置静态网站托管：

1. 进入 Bucket >「基础设置」>「静态页面」
2. 设置默认首页：`index.html`
3. 设置 404 页面：`404.html`（可选）
4. 保存配置

配置后，访问 `https://yourdomain.com/` 会自动返回 `index.html` 的内容。

### 域名备案替代方案

如果域名无法备案：
1. **使用海外地域 Bucket**：中国香港、新加坡等地域不要求域名备案
2. **使用 CDN**：通过 CDN 加速域名访问 OSS，CDN 域名需要备案
3. **等待备案完成**：ICP 备案通常需要 3-20 个工作日

## 常见问题

### Q1：为什么配置了静态网站托管但仍然下载？

检查您是否通过 OSS 默认域名访问。静态网站托管功能**必须配合自定义域名**使用，OSS 默认域名始终是下载行为。

### Q2：微服务前端项目的路由怎么配？

对于 Vue/React 等 SPA 单页应用：
1. 将所有前端构建产物（如 `dist/` 目录）上传到 OSS
2. 静态网站托管中设置默认首页为 `index.html`
3. 设置 404 页面也为 `index.html`（让前端路由处理所有路径）

### Q3：访问根路径 URL 会变成下载 index.html？

确认：
1. 是否通过自定义域名访问（OSS 默认域名会下载）
2. 静态网站托管是否正确配置了默认首页
3. index.html 文件的 Content-Type 是否为 `text/html`（上传时检查）

### Q4：OSS 拒绝 HTTP 访问配置

如果要求强制 HTTPS 访问：
1. 在 Bucket 的「数据安全」>「TLS版本设置」中配置
2. 为自定义域名配置 SSL 证书
