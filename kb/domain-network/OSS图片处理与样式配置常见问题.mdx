# OSS 图片处理与样式配置常见问题

## Q1：如何通过 URL 压缩/缩放图片？

在图片 URL 后添加 `x-oss-process` 参数：

```
# 按宽度等比缩放到 800px
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=image/resize,w_800

# 按高度等比缩放到 600px
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=image/resize,h_600

# 同时指定宽高，按比例缩放（不裁剪）
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=image/resize,m_lfit,w_800,h_600

# 压缩质量到 80%
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=image/quality,q_80
```

## Q2：如何使用自定义分隔符简化图片处理 URL？

OSS 支持使用样式分隔符替代 `x-oss-process=style/`：

```
# 标准方式
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=style/thumbnail

# 使用分隔符 "!"
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg!thumbnail
```

**配置步骤**：
1. 在 OSS 控制台进入 Bucket >「数据处理」>「图片处理」
2. 创建样式（如 `thumbnail`），定义处理参数
3. 在「访问设置」中设置分隔符（如 `!`）

> **注意**：分隔符只能用于已创建的样式名，不能用于直接写图片处理参数。

## Q3：图片处理后的 Content-Type 变成 application/xml？

这种情况通常是图片处理参数错误或原图不存在，OSS 返回了 XML 格式的错误信息。排查步骤：
1. 先访问原图 URL（去掉 `x-oss-process` 参数）确认文件存在
2. 检查图片处理参数语法是否正确
3. 确认原图格式是 OSS 支持的图片格式（jpg、png、gif、webp、bmp、tiff）

## Q4：开启原图保护后图片无法访问？

开启原图保护后，不能通过 OSS URL 直接访问原图，必须通过图片处理参数或样式访问：

```
# 被拒绝（原图保护生效）
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg

# 正常访问（带处理参数）
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=image/resize,w_800

# 正常访问（使用样式）
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg!thumbnail
```

## Q5：如何获取图片处理后的宽高信息？

使用 `image/info` 参数：
```
https://bucket.oss-cn-xxx.aliyuncs.com/image.jpg?x-oss-process=image/info
```

返回 JSON 格式的图片元信息，包括宽度、高度、格式等。

## Q6：OSS 控制台能直接批量压缩图片吗？

不能。OSS 控制台没有批量压缩功能。实现方式：
1. **实时处理**：通过 URL 参数实时处理，不修改原图
2. **持久化处理**：使用图片处理持久化功能，将处理后的图片保存为新文件（可覆盖原图）
3. **批量处理**：需要编写代码配合 SDK 实现

图片处理持久化示例（覆盖原图）：
```
x-oss-process=image/resize,w_800|sys/saveas,o_aW1hZ2UuanBn,b_YnVja2V0
```

参考文档：[保存处理后的图片](https://help.aliyun.com/zh/oss/user-guide/save-processed-images)

## Q7：GIF 动图处理后变成静态图？

默认情况下，OSS 对 GIF 进行 resize 等操作时只处理第一帧。如需保持动画效果，需要确认处理参数的兼容性，部分操作不支持 GIF 动画保持。
