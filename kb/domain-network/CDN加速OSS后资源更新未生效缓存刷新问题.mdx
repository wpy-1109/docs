# CDN 加速 OSS 后资源更新未生效（缓存刷新问题）

## 症状

通过 CDN 域名访问 OSS 资源时，发现：
- OSS 上的文件已更新（如替换了同名图片），但通过 CDN 域名访问仍显示旧内容
- 多个 CDN 域名指向同一个 OSS 资源，刷新一个域名后其他域名仍是旧版本
- OSS 图片处理规则已修改，但 CDN 返回的图片样式未变

## 原因

CDN 会在边缘节点缓存 OSS 的资源。当 OSS 源站文件更新后，CDN 节点的缓存不会自动失效，用户访问时仍会命中旧缓存。

## 解决方案

### 方案 1：手动刷新 CDN 缓存

1. 登录 [CDN 控制台](https://cdn.console.aliyun.com/)
2. 在左侧导航栏选择「刷新预热」
3. 选择「刷新」，输入需要刷新的 URL 或目录
4. 提交刷新任务

**刷新类型选择**：
| 类型 | 适用场景 |
|------|---------|
| URL 刷新 | 单个文件更新，精确刷新 |
| 目录刷新 | 批量文件更新，刷新整个目录 |

### 方案 2：多个 CDN 域名需分别刷新

当多个 CDN 域名指向同一个 OSS 资源时，CDN 缓存是按加速域名隔离的。每个域名需要单独提交刷新请求，不支持刷新一次让所有域名同时生效。

### 方案 3：使用文件版本号避免缓存问题

在 URL 中添加版本号参数，每次更新文件时修改版本号：

```
https://cdn.yourdomain.com/image.jpg?v=20260301
# 更新后改为
https://cdn.yourdomain.com/image.jpg?v=20260302
```

这样 CDN 会将其视为新 URL，自动回源获取最新内容。

### 方案 4：调整 CDN 缓存策略

在 CDN 控制台设置更短的缓存过期时间：
1. 进入域名管理 > 缓存配置
2. 针对频繁更新的文件类型设置较短的缓存时间（如 0 秒或 300 秒）

### CDN URL 鉴权相关问题

如果 CDN 开启了 URL 鉴权（如 C 类鉴权），需注意：
- 访问 URL 必须携带鉴权参数才能正常访问
- 鉴权规则中的文件匹配条件需正确配置：例如匹配 `.mp4` 文件应写 `.mp4` 而非 `*.mp4`
- 未通过鉴权的请求会返回 403 错误

## 预防建议

- 对于频繁更新的资源，建议在 URL 中加版本号或时间戳
- 大规模文件更新前，提前准备好刷新任务
- 可使用 CDN API 自动化刷新操作
