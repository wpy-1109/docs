# OSS 文件下载时文件名乱码或丢失的解决方案

## 症状

- 下载 OSS 文件时，中文文件名显示为乱码
- 苹果手机微信内下载 OSS 链接时文件名显示为 `null`
- 之前正常的中文下载名突然变成乱码
- 不同浏览器/设备上文件名表现不一致

## 原因

浏览器对 HTTP 响应头中 `Content-Disposition` 的 `filename` 参数解析存在差异：
- 直接在 `filename` 中使用中文字符，部分浏览器/设备无法正确解析
- 不同浏览器对字符编码的处理不同（UTF-8、GBK 等）
- iOS 微信内置浏览器对文件名的处理与标准浏览器不同

## 解决方案

### 方案 1：使用 RFC 5987 标准的 filename* 参数

设置 `Content-Disposition` 时同时提供 `filename` 和 `filename*`：

```
Content-Disposition: attachment; filename="document.pdf"; filename*=UTF-8''%E4%B8%AD%E6%96%87%E6%96%87%E6%A1%A3.pdf
```

说明：
- `filename`：ASCII 备用名，给不支持 `filename*` 的旧浏览器用
- `filename*=UTF-8''...`：UTF-8 编码的文件名，`''` 后是 URL 编码的中文文件名

### 方案 2：在 OSS 中设置 Content-Disposition

**方法一：上传时设置 Header**

通过 SDK 上传时指定 `Content-Disposition`：

```python
# Python SDK 示例
bucket.put_object('path/file.pdf', data,
    headers={'Content-Disposition': "attachment; filename=\"doc.pdf\"; filename*=UTF-8''%E4%B8%AD%E6%96%87.pdf"})
```

**方法二：在 OSS 控制台修改 Object 的 HTTP Header**

1. 进入 Bucket 找到目标文件
2. 点击文件详情 > 设置 HTTP 头
3. 在 Content-Disposition 中填写正确的值

**方法三：通过签名 URL 指定**

在生成签名 URL 时通过 `response-content-disposition` 参数指定：

```
https://bucket.oss-cn-xxx.aliyuncs.com/file.pdf?response-content-disposition=attachment%3B%20filename%3D%22doc.pdf%22%3B%20filename*%3DUTF-8''%25E4%25B8%25AD%25E6%2596%2587.pdf
```

### 方案 3：URL 编码中文文件名

对中文文件名进行 URL 编码（percent-encoding）：

```python
import urllib.parse
filename = "中文文档.pdf"
encoded = urllib.parse.quote(filename)
# 结果: %E4%B8%AD%E6%96%87%E6%96%87%E6%A1%A3.pdf
```

## 平台兼容性参考

| 平台/浏览器 | filename | filename* | 推荐 |
|------------|----------|-----------|------|
| Chrome/Edge | 支持 UTF-8 | 支持 | 使用 filename* |
| Firefox | 支持 UTF-8 | 支持 | 使用 filename* |
| Safari | 部分支持 | 支持 | 使用 filename* |
| iOS 微信 | 可能显示 null | 可能不生效 | 使用 URL 编码 |
| Android 微信 | 支持 | 支持 | 使用 filename* |

## 最佳实践

```
Content-Disposition: attachment; filename="fallback-name.pdf"; filename*=UTF-8''%E5%AE%9E%E9%99%85%E6%96%87%E4%BB%B6%E5%90%8D.pdf
```

同时提供两个字段，确保最大兼容性。
