# OSS 跨区域复制配置与进度异常排查

## 适用场景

- 同账号下不同地域的 Bucket 之间同步数据（如中国区到新加坡区）
- 同账号同地域的 Bucket 之间复制数据
- 跨账号的 Bucket 数据复制
- 因地域关停需要迁移 Bucket 数据

## 症状

- 跨区域复制进度一直卡在 99%，不到 100%
- 跨区域复制任务配置后目标 Bucket 没有文件
- 开启版本控制后无法创建跨区域复制任务（按钮灰色）
- 地域关停后无法配置跨区域复制

## 按问题排查

### 问题 1：复制进度卡在 99% 不动

**原因**：历史数据复制进度并非实时更新，需要等待所有文件扫描完成后才会刷新。如果 Bucket 内文件数量庞大（百万或亿级），进度更新可能需要数小时。

**确认方法**：
1. 查看目标 Bucket 的存储容量是否在增长
2. 查看复制流入流量监控是否有数据传入
3. 对比源和目标 Bucket 的文件总量和总大小

> **注意**：进度显示 99% 并不意味着数据未在复制，只是进度条更新延迟。通常等待即可。

### 问题 2：跨区域复制任务已完成但目标 Bucket 没有文件

**排查步骤**：
1. 确认操作方向是否正确：跨区域复制需要**在源 Bucket 上配置**，不是在目标 Bucket 上
2. 确认是否勾选了「复制历史数据」选项
3. 确认源 Bucket 中确实有文件（检查文件路径前缀是否正确）

### 问题 3：开启版本控制后创建跨区域复制按钮变灰

**原因**：如果源 Bucket 开启了版本控制，目标 Bucket 也必须开启版本控制，否则无法创建复制任务。

**解决方案**：先对目标 Bucket 开启版本控制，然后再配置跨区域复制。

### 问题 4：地域关停后无法配置跨区域复制

已关停或关停中的地域（如华东5南京、华东6福州本地地域）可能不再支持新建复制任务。

**替代方案**：
1. 使用 ossutil 命令行工具手动迁移：
```bash
# 从源 Bucket 下载到本地
ossutil cp -r oss://source-bucket/ /local/path/ --endpoint oss-cn-nanjing.aliyuncs.com

# 上传到目标 Bucket
ossutil cp -r /local/path/ oss://dest-bucket/ --endpoint oss-cn-hangzhou.aliyuncs.com
```
2. 使用阿里云[在线迁移服务](https://help.aliyun.com/zh/oss/user-guide/migrate-data-by-using-the-online-migration-service)进行托管迁移

### 问题 5：跨区域复制是否走内网？

不走内网。跨区域复制走阿里云骨干网络（公网），会产生跨区域复制流量费用。同区域内的复制走内网，不产生外网流量费用。

## 跨区域复制配置要点

### 性能指标

| 场景 | 复制时效 |
|------|---------|
| 开启 RTC（数据复制时间控制） | 几秒内复制大多数 Object，10 分钟内复制 99.99% |
| 未开启 RTC | 几分钟到几小时，取决于数据量 |

### 配置前须知

1. 源和目标 Bucket 的版本控制状态必须一致（都开启或都不开启）
2. 同一个 Bucket 的同一前缀只能配置一条复制规则
3. 复制完成后如需删除源 Bucket，**必须先关闭复制任务**，否则可能导致数据不一致
4. 复制是异步的，370GB 数据可能需要数小时

### 配置步骤

1. 进入源 Bucket >「数据管理」>「跨区域复制」（或「同区域复制」）
2. 点击「创建规则」
3. 选择目标区域和目标 Bucket
4. 选择复制范围（整个 Bucket 或指定前缀）
5. 选择是否「复制历史数据」
6. 选择是否启用 RTC（推荐，可加速复制）
7. 确认并创建

## 常见问题

### Q1：Bucket 可以直接改地域吗？

不可以。OSS Bucket 的地域在创建后无法修改。如需更换地域，只能创建新 Bucket 并将数据迁移过去。

### Q2：跨区域复制完成后需要手动关闭吗？

如果只是做一次性数据迁移，复制完成后建议关闭复制任务。如果保持开启，后续源 Bucket 的增删改操作会持续同步到目标 Bucket（包括删除操作）。

### Q3：跨账号的数据复制怎么做？

跨账号支持跨区域复制，但需要在目标账号创建 RAM 角色并授权源账号访问。参考[跨账号跨区域复制](https://help.aliyun.com/zh/oss/user-guide/cross-region-replication-across-accounts)。
