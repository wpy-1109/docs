Configure Alibaba Cloud CDN to cache and distribute static resources from your OSS bucket to globally distributed edge nodes. This reduces access latency, lowers OSS outbound traffic costs, and improves user experience.

## How it works

1. A user requests a resource. Smart DNS routes the request to the nearest CDN point of presence (POP).
2. If the POP has the resource cached, it serves it directly.
3. If not, the POP fetches it from your OSS bucket (origin), caches it, and serves the response.
4. Subsequent requests for the same resource are served from the POP cache.

## Prerequisites

- A registered domain name. If the acceleration region is in the Chinese mainland, the domain must have an ICP filing.
- An OSS bucket with the resources you want to accelerate.

## Configure CDN acceleration

<Steps>
  <Step title="Add an accelerated domain name">
    Go to the [CDN console](https://cdnnext.console.aliyun.com/domain/list), click **Add Domain Name**, select an acceleration region and business type, and set the origin URL to your OSS bucket domain.
  </Step>
  <Step title="Configure the CNAME record">
    Copy the CNAME address assigned by CDN. In your DNS console, add a CNAME record pointing your accelerated domain to this address.
  </Step>
  <Step title="Configure private bucket access">
    If your bucket is private, go to **Origin Fetch** in the CDN console and enable **Alibaba Cloud OSS Private Bucket Access**.
  </Step>
  <Step title="Verify the acceleration">
    Access a resource through your accelerated domain and check the `X-Cache` response header. `HIT` indicates the CDN cache was used.
  </Step>
</Steps>

<Warning>
After enabling private bucket access, CDN automatically signs origin fetch requests. Clients must use URLs **without** signature parameters. Including signature parameters causes a double-signing conflict and a 403 error.
</Warning>

## Best practices

- **Enable HTTPS** on the accelerated domain to encrypt data between clients and CDN POPs.
- **Set cache TTLs** by resource type: long TTLs for images and videos, shorter TTLs for JS/CSS, and no caching for dynamic content.
- **Enable Auto CDN Cache Update** in OSS to automatically refresh CDN caches when you update objects.
- **Configure CORS headers on CDN** directly, not just at the OSS origin, to ensure cross-origin headers are included in cached responses.

## Billing

| Component | Description |
|-----------|-------------|
| CDN traffic | Billed for traffic served from CDN POPs to users |
| OSS origin fetch | Billed for traffic from OSS to CDN on cache misses |
