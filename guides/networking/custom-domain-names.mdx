Map a custom domain name to your bucket to access OSS resources using your own domain instead of the default endpoint. This enables online file preview, improves brand consistency, and supports HTTPS certificate configuration.

## How it works

Mapping uses a CNAME (Canonical Name) DNS record. You point your custom domain to the OSS bucket's default domain name, and DNS resolves requests to the corresponding OSS endpoint.

| Scenario | CNAME Target |
|----------|-------------|
| Public access | Bucket public endpoint (e.g., `examplebucket.oss-cn-hangzhou.aliyuncs.com`) |
| Transfer acceleration | Acceleration endpoint (e.g., `examplebucket.oss-accelerate.aliyuncs.com`) |

## Prerequisites

- A registered domain name. If the bucket is in a Chinese mainland region, the domain must have an ICP filing.
- A maximum of 100 custom domain names can be mapped per bucket.

## Using the console

<Steps>
  <Step title="Map the domain name">
    Go to **Bucket Settings > Domain Names** and click **Map Custom Domain Name**. Enter your domain name and verify ownership if prompted.
  </Step>
  <Step title="Configure the CNAME record">
    In your DNS provider's console, add a CNAME record pointing your domain to the bucket's public endpoint. If the domain belongs to the current Alibaba Cloud account, select **Automatically Add CNAME Record**.
  </Step>
  <Step title="Verify access">
    For public-read buckets, access an object directly: `http://image.example.com/example.jpg`. For private buckets, use a presigned URL with the custom domain.
  </Step>
</Steps>

<Warning>
Complete the domain mapping in OSS before configuring the CNAME record. Otherwise, OSS cannot identify the target bucket, and requests return unexpected content.
</Warning>

## Quotas and limits

- Up to 100 custom domain names per bucket.
- Each domain can map to only one bucket at a time.
- Wildcard domains (`*.example.com`) and Chinese domain names are not supported.
