Object FC Access Points let you attach a Function Compute function to an access point. When a client reads an object through the access point, OSS automatically invokes the function to transform the data before returning it. Use cases include watermarking images, redacting sensitive data, and converting file formats.

## How it works

1. Client sends a GetObject request to the Object FC Access Point.
2. OSS invokes the configured Function Compute function.
3. The function processes the original object data.
4. The function calls `WriteGetObjectResponse` to return the processed data.
5. The client receives the transformed data.

## Prerequisites

- An OSS bucket with an access point
- A Function Compute function deployed in the same region
- The `fc:InvokeFunction` permission

## Using the console

<Steps>
  <Step title="Create an Object FC Access Point">
    Navigate to your bucket, choose **Access Point**, then click **Create Object FC Access Point**.
  </Step>
  <Step title="Configure function">
    Select the Function Compute service and function to invoke when objects are accessed.
  </Step>
  <Step title="Set access policy">
    Attach a policy to control who can access objects through this access point.
  </Step>
</Steps>
