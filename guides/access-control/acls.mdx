Access control lists (ACLs) provide a basic level of access control for buckets and objects. While RAM policies and bucket policies offer more granular control, ACLs are useful for simple public/private access settings.

## Bucket ACL levels

| ACL | Description |
|-----|-------------|
| **private** | Only the bucket owner can read and write. Default setting. |
| **public-read** | Anyone can read, only the owner can write. |
| **public-read-write** | Anyone can read and write. Not recommended. |

## Object ACL levels

| ACL | Description |
|-----|-------------|
| **default** | Inherits the bucket ACL. |
| **private** | Only the object owner can read and write. |
| **public-read** | Anyone can read. |
| **public-read-write** | Anyone can read and write. |

<Warning>
Avoid using `public-read-write` ACLs. Use bucket policies or presigned URLs instead for controlled access.
</Warning>

## Using an SDK

<Tabs>
  <Tab title="Python">
    ```python title="set_acl.py"
    import alibabacloud_oss_v2 as oss

    credentials_provider = oss.credentials.EnvironmentVariableCredentialsProvider()
    cfg = oss.config.load_default()
    cfg.credentials_provider = credentials_provider
    cfg.region = "cn-hangzhou"

    client = oss.Client(cfg)

    # Set bucket ACL
    client.put_bucket_acl(oss.PutBucketAclRequest(
        bucket="examplebucket",
        acl="private",
    ))

    # Set object ACL
    client.put_object_acl(oss.PutObjectAclRequest(
        bucket="examplebucket",
        key="example.txt",
        acl="public-read",
    ))
    ```
  </Tab>
  <Tab title="Java">
    ```java title="SetACL.java"
    // Set bucket ACL
    ossClient.setBucketAcl("examplebucket", CannedAccessControlList.Private);

    // Set object ACL
    ossClient.setObjectAcl("examplebucket", "example.txt",
        CannedAccessControlList.PublicRead);
    ```
  </Tab>
</Tabs>
