Resource Access Management (RAM) policies define which users can perform which actions on which OSS resources. You attach RAM policies to RAM users, groups, or roles to implement fine-grained access control.

## Policy structure

A RAM policy is a JSON document with the following structure:

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:GetObject", "oss:PutObject"],
      "Resource": ["acs:oss:*:*:examplebucket/*"],
      "Condition": {}
    }
  ]
}
```

| Field | Description |
|-------|-------------|
| `Effect` | `Allow` or `Deny` |
| `Action` | OSS operations (e.g., `oss:GetObject`, `oss:PutObject`, `oss:*`) |
| `Resource` | ARN of the OSS resource (`acs:oss:*:*:bucket-name/key-prefix*`) |
| `Condition` | Optional conditions (IP, SSL, tags, etc.) |

## Common policy examples

### Read-Only Access to a bucket

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:ListObjects", "oss:GetObject"],
      "Resource": [
        "acs:oss:*:*:examplebucket",
        "acs:oss:*:*:examplebucket/*"
      ]
    }
  ]
}
```

### Read-Write Access to a specific prefix

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:GetObject", "oss:PutObject", "oss:DeleteObject"],
      "Resource": ["acs:oss:*:*:examplebucket/user-data/*"]
    },
    {
      "Effect": "Allow",
      "Action": ["oss:ListObjects"],
      "Resource": ["acs:oss:*:*:examplebucket"],
      "Condition": {
        "StringLike": {
          "oss:Prefix": "user-data/*"
        }
      }
    }
  ]
}
```

### Full Access

Attach the system policy `AliyunOSSFullAccess` for complete OSS access, or `AliyunOSSReadOnlyAccess` for read-only access.

## Best practices

<Tip>
Follow the principle of least privilege: grant only the minimum permissions required for each user or role.
</Tip>

- Use separate RAM users for each application or team member.
- Prefer STS temporary credentials over long-lived AccessKeys.
- Use conditions to restrict access by IP address, SSL, or time.
- Regularly audit and rotate credentials.
