Bucket policies are resource-based policies attached directly to a bucket. They allow you to grant cross-account access, restrict access by IP address, or enforce HTTPS without modifying RAM policies.

## When to use bucket policies

- Grant access to users from other Alibaba Cloud accounts
- Restrict access by source IP address or VPC
- Enforce HTTPS-only access
- Allow anonymous read access to specific prefixes

## Policy example

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": ["*"],
      "Action": ["oss:GetObject"],
      "Resource": ["acs:oss:*:*:examplebucket/public/*"],
      "Condition": {
        "StringEquals": {
          "oss:SecureTransport": "true"
        }
      }
    }
  ]
}
```

## Using the console

<Steps>
  <Step title="Navigate to Bucket Policy">
    Log on to the [OSS console](https://oss.console.alibabacloud.com/), click the bucket, then choose **Permission Control > Bucket Policy**.
  </Step>
  <Step title="Add a policy">
    Click **Authorize** on the GUI tab to configure principals, actions, resources, and conditions visually.
  </Step>
</Steps>
