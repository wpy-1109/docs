Security Token Service (STS) provides temporary credentials with limited permissions and a defined expiration time. Use STS to grant mobile apps, web browsers, or third-party users secure access to OSS without sharing long-lived AccessKeys.

## How it works

1. Your application server calls the STS `AssumeRole` API with a RAM role ARN and session policy.
2. STS returns temporary credentials: AccessKey ID, AccessKey Secret, and Security Token.
3. The client uses these credentials to access OSS directly.
4. Credentials expire automatically after the specified duration (15 minutes to 12 hours).

## Prerequisites

- SDK installed and configured ([quickstart](/quickstart))
- A RAM role with OSS permissions
- The `sts:AssumeRole` permission

## Get temporary credentials

<Tabs>
  <Tab title="Python">
    ```python title="sts_credentials.py"
    from alibabacloud_sts20150401.client import Client as StsClient
    from alibabacloud_tea_openapi.models import Config
    from alibabacloud_sts20150401.models import AssumeRoleRequest
    import os

    config = Config(
        access_key_id=os.environ["OSS_ACCESS_KEY_ID"],
        access_key_secret=os.environ["OSS_ACCESS_KEY_SECRET"],
        endpoint="sts.aliyuncs.com",
    )

    sts_client = StsClient(config)

    request = AssumeRoleRequest(
        role_arn="acs:ram::123456789:role/oss-readonly",
        role_session_name="app-session",
        duration_seconds=3600,
    )

    response = sts_client.assume_role(request)
    credentials = response.body.credentials

    print(f"AccessKeyId: {credentials.access_key_id}")
    print(f"AccessKeySecret: {credentials.access_key_secret}")
    print(f"SecurityToken: {credentials.security_token}")
    print(f"Expiration: {credentials.expiration}")
    ```
  </Tab>
  <Tab title="Java">
    ```java title="StsCredentials.java"
    import com.aliyuncs.DefaultAcsClient;
    import com.aliyuncs.profile.DefaultProfile;
    import com.aliyuncs.sts.model.v20150401.*;

    DefaultProfile profile = DefaultProfile.getProfile("cn-hangzhou",
        System.getenv("OSS_ACCESS_KEY_ID"),
        System.getenv("OSS_ACCESS_KEY_SECRET"));
    DefaultAcsClient client = new DefaultAcsClient(profile);

    AssumeRoleRequest request = new AssumeRoleRequest();
    request.setRoleArn("acs:ram::123456789:role/oss-readonly");
    request.setRoleSessionName("app-session");
    request.setDurationSeconds(3600L);

    AssumeRoleResponse response = client.getAcsResponse(request);
    AssumeRoleResponse.Credentials credentials = response.getCredentials();

    System.out.println("AccessKeyId: " + credentials.getAccessKeyId());
    System.out.println("SecurityToken: " + credentials.getSecurityToken());
    ```
  </Tab>
</Tabs>

## Use STS credentials with OSS

```python
import alibabacloud_oss_v2 as oss

cfg = oss.config.load_default()
cfg.credentials_provider = oss.credentials.StaticCredentialsProvider(
    access_key_id=credentials.access_key_id,
    access_key_secret=credentials.access_key_secret,
    security_token=credentials.security_token,
)
cfg.region = "cn-hangzhou"

client = oss.Client(cfg)
# Use client normally -- credentials expire automatically
```

<Warning>
Never expose STS credentials in client-side code. Return them from your server via a secure API endpoint.
</Warning>
