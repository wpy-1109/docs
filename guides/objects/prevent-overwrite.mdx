By default, uploading an object with the same key overwrites the existing object. You can use the `x-oss-forbid-overwrite` header to prevent accidental overwrites.

## Using the forbid overwrite header

Set the `x-oss-forbid-overwrite` header to `true` in your upload request. If an object with the same key already exists, OSS returns a `409 Conflict` error instead of overwriting.

<Tabs>
  <Tab title="Python">
    ```python title="prevent_overwrite.py"
    import alibabacloud_oss_v2 as oss

    credentials_provider = oss.credentials.EnvironmentVariableCredentialsProvider()
    cfg = oss.config.load_default()
    cfg.credentials_provider = credentials_provider
    cfg.region = "cn-hangzhou"

    client = oss.Client(cfg)

    result = client.put_object(oss.PutObjectRequest(
        bucket="examplebucket",
        key="important-file.txt",
        body=b"File contents",
        forbid_overwrite=True,
    ))
    ```
  </Tab>
  <Tab title="Java">
    ```java title="PreventOverwrite.java"
    PutObjectRequest request = new PutObjectRequest("examplebucket",
        "important-file.txt", new ByteArrayInputStream("File contents".getBytes()));
    ObjectMetadata metadata = new ObjectMetadata();
    metadata.setHeader("x-oss-forbid-overwrite", "true");
    request.setMetadata(metadata);
    ossClient.putObject(request);
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript title="preventOverwrite.js"
    await client.put('important-file.txt', Buffer.from('File contents'), {
      headers: { 'x-oss-forbid-overwrite': 'true' },
    });
    ```
  </Tab>
</Tabs>

## Other protection methods

| Method | Description |
|--------|-------------|
| **Versioning** | Keeps all previous versions of an object. Overwrites create new versions. |
| **WORM retention** | Prevents modification or deletion for a specified retention period. |
| **RAM policies** | Deny `oss:PutObject` to prevent specific users from overwriting. |
