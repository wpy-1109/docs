A symbolic link (symlink) is a special object that points to another object in the same bucket. You can access the target object through the symlink, similar to shortcuts in a file system. Symlinks allow you to create multiple access paths to the same data without duplicating it.

## Prerequisites

- The `oss:PutObject` permission (to create symlinks)
- The `oss:GetObject` permission (to read symlinks)

## Limitations

- The target object must exist in the same bucket.
- Symlinks cannot point to other symlinks.
- Symlinks do not support range reads or append operations.

## Using an SDK

<Tabs>
  <Tab title="Python">
    ```python title="symbolic_links.py"
    import alibabacloud_oss_v2 as oss

    credentials_provider = oss.credentials.EnvironmentVariableCredentialsProvider()
    cfg = oss.config.load_default()
    cfg.credentials_provider = credentials_provider
    cfg.region = "cn-hangzhou"

    client = oss.Client(cfg)

    # Create a symlink
    client.put_symlink(oss.PutSymlinkRequest(
        bucket="examplebucket",
        key="shortcut.txt",
        target="original/path/file.txt",
    ))

    # Read the symlink target
    result = client.get_symlink(oss.GetSymlinkRequest(
        bucket="examplebucket",
        key="shortcut.txt",
    ))

    print(f"Target: {result.target}")
    ```
  </Tab>
  <Tab title="Java">
    ```java title="SymbolicLinks.java"
    // Create a symlink
    ossClient.createSymlink("examplebucket", "shortcut.txt", "original/path/file.txt");

    // Get symlink target
    OSSSymlink symlink = ossClient.getSymlink("examplebucket", "shortcut.txt");
    System.out.println("Target: " + symlink.getTarget());
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript title="symbolicLinks.js"
    // Create a symlink
    await client.putSymlink('shortcut.txt', 'original/path/file.txt');

    // Get symlink target
    const result = await client.getSymlink('shortcut.txt');
    console.log('Target:', result.res.headers['x-oss-symlink-target']);
    ```
