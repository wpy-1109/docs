Image styles let you save a set of image processing parameters as a named style. Instead of appending long processing strings to every URL, reference the style name for consistent, reusable transformations.

## How it works

1. Create a named style with a combination of processing operations (e.g., resize + quality + format).
2. Reference the style in your image URLs using the style separator.

```
https://examplebucket.oss-cn-hangzhou.aliyuncs.com/photo.jpg?x-oss-process=style/thumbnail
```

## Create a style

<Steps>
  <Step title="Open image styles">
    Navigate to your bucket, choose **Data Processing > Image Processing**, and click the **Styles** tab.
  </Step>
  <Step title="Create a new style">
    Click **Create Style**, enter a name (e.g., `thumbnail`), and configure the processing operations using the visual editor or manual parameters.
  </Step>
  <Step title="Save and use">
    Save the style. Use it in URLs with `?x-oss-process=style/<style-name>`.
  </Step>
</Steps>

## Style separator

You can configure a custom style separator to simplify URLs. For example, with a separator of `-`:

```
https://examplebucket.oss-cn-hangzhou.aliyuncs.com/photo.jpg-thumbnail
```

<Warning>
Choose a style separator that does not conflict with characters in your object keys. Common separators include `-`, `_`, `!`, and `/`.
</Warning>

## Example styles

| Style Name | Operations | Use Case |
|-----------|-----------|----------|
| `thumbnail` | `image/resize,w_200/quality,q_80` | List page thumbnails |
| `preview` | `image/resize,w_800/format,webp` | Image preview |
| `avatar` | `image/resize,w_100,h_100,m_fill/circle,r_50` | User avatars |

## Limits

- Up to 50 styles per bucket.
- Style names can contain letters, digits, hyphens, and underscores.
