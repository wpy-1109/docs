Lifecycle rules automate storage management by transitioning objects to cheaper storage classes or deleting them after a specified period. This reduces costs and eliminates manual cleanup.

## Rule types

| Action | Description |
|--------|-------------|
| **Transition** | Move objects to a different storage class (e.g., Standard to IA after 30 days). |
| **Expiration** | Delete objects after a specified number of days. |
| **Abort multipart upload** | Clean up incomplete multipart uploads after a specified period. |
| **Delete markers** | Remove expired delete markers in versioned buckets. |

## Using the console

<Steps>
  <Step title="Navigate to Lifecycle">
    Log on to the OSS console, click your bucket, then choose **Data Management > Lifecycle**.
  </Step>
  <Step title="Create a rule">
    Click **Create Rule**, configure the scope (entire bucket or prefix), actions, and timing.
  </Step>
</Steps>

## Using an SDK

<Tabs>
  <Tab title="Python">
    ```python title="lifecycle_rules.py"
    import alibabacloud_oss_v2 as oss

    credentials_provider = oss.credentials.EnvironmentVariableCredentialsProvider()
    cfg = oss.config.load_default()
    cfg.credentials_provider = credentials_provider
    cfg.region = "cn-hangzhou"

    client = oss.Client(cfg)

    client.put_bucket_lifecycle(oss.PutBucketLifecycleRequest(
        bucket="examplebucket",
        lifecycle_configuration=oss.LifecycleConfiguration(
            rules=[
                oss.LifecycleRule(
                    id="transition-to-ia",
                    prefix="logs/",
                    status="Enabled",
                    transitions=[
                        oss.LifecycleRuleTransition(
                            days=30,
                            storage_class="IA",
                        )
                    ],
                    expiration=oss.LifecycleRuleExpiration(days=365),
                )
            ]
        )
    ))
    ```
  </Tab>
</Tabs>
