Direct client upload lets web browsers and mobile apps upload files directly to OSS without routing data through your application server. This reduces server load, lowers latency, and simplifies your architecture.

## How it works

1. The client requests upload credentials from your application server.
2. Your server generates STS temporary credentials or a presigned URL and returns them to the client.
3. The client uploads the file directly to OSS using the credentials.
4. Optionally, OSS sends an [upload callback](/guides/objects/upload-callback) to your server to notify it of the completed upload.

## Methods

| Method | Description | Best For |
|--------|-------------|---------|
| STS credentials | Temporary AK/SK + security token | SDK-based uploads with fine-grained control |
| Presigned URL | Pre-authorized upload URL | Simple single-file uploads |
| POST policy | Signed form policy for HTML form uploads | Browser-based form submissions |

## Using STS credentials

<Steps>
  <Step title="Server: Generate STS credentials">
    Your server calls STS AssumeRole to generate temporary credentials scoped to the target bucket and prefix.
  </Step>
  <Step title="Client: Upload with SDK">
    The client initializes the OSS SDK with the temporary credentials and uploads the file.
  </Step>
</Steps>

## Using a presigned URL

<Steps>
  <Step title="Server: Generate presigned URL">
    Your server generates a presigned PUT URL with an expiration time.
  </Step>
  <Step title="Client: Upload with HTTP PUT">
    The client sends an HTTP PUT request to the presigned URL with the file as the request body.
  </Step>
</Steps>

```javascript
// Client-side upload using fetch
const response = await fetch(presignedUrl, {
  method: "PUT",
  body: file,
  headers: { "Content-Type": file.type },
});
```

## Security best practices

- **Scope credentials** to the minimum required permissions (specific bucket, prefix, and actions).
- **Set short expiration times** for STS tokens and presigned URLs (15 minutes to 1 hour).
- **Validate file types and sizes** on both client and server side.
- **Use upload callbacks** to verify and process uploaded files.
