Elastic File Cache (EFC) provides a high-performance caching layer between your compute instances and OSS. It caches frequently accessed data on local or distributed storage close to your compute resources, reducing read latency and improving throughput for data-intensive workloads.

## When to use

- **Machine learning training**: Cache training datasets to avoid repeated downloads from OSS.
- **Data analytics**: Speed up iterative queries over the same dataset.
- **Media processing**: Cache source media files during transcoding and rendering pipelines.
- **HPC simulations**: Reduce I/O bottlenecks for simulation input/output data.

## How it works

1. EFC mounts as a POSIX-compatible file system on your compute instances.
2. On first read, data is fetched from OSS and cached locally.
3. Subsequent reads are served from the cache with low latency.
4. Writes can be configured to write-through (sync to OSS immediately) or write-back (batch sync).

## Configure EFC

<Steps>
  <Step title="Create an EFC file system">
    In the [EFC console](https://nasnext.console.alibabacloud.com/), create a cache file system. Select the region matching your compute resources.
  </Step>
  <Step title="Associate with OSS">
    Configure the data source as your OSS bucket and specify the prefix to cache.
  </Step>
  <Step title="Mount on compute instances">
    Install the EFC client on your ECS instances and mount the cache file system.
  </Step>
</Steps>

## Performance characteristics

| Metric | OSS Direct | EFC Cached |
|--------|-----------|------------|
| Read latency | 10-50 ms | Sub-millisecond |
| Throughput | Limited by network | Local storage speed |
| POSIX support | Limited (SDK/API) | Full POSIX file system |
