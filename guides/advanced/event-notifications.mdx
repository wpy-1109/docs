Event notifications send messages to downstream services when specific events occur in your bucket, such as object creation, deletion, or replication completion. Use event notifications to build event-driven architectures and automate workflows.

## Supported events

| Event Type | Trigger |
|-----------|---------|
| `ObjectCreated:*` | Any object creation (PutObject, PostObject, CopyObject, CompleteMultipartUpload) |
| `ObjectCreated:PutObject` | Simple upload |
| `ObjectCreated:PostObject` | Form-based upload |
| `ObjectRemoved:*` | Any object deletion |
| `ObjectRemoved:DeleteObject` | Single object deletion |
| `ObjectReplication:*` | Cross-region or same-region replication events |

## Notification targets

| Target | Description |
|--------|-------------|
| Message Service (MNS) | Queue or topic for asynchronous processing |
| Function Compute | Serverless function execution |
| EventBridge | Central event bus for routing to multiple targets |

## Configure event notifications

<Steps>
  <Step title="Open event notification settings">
    Navigate to your bucket, choose **Event Notification**, and click **Create Rule**.
  </Step>
  <Step title="Define the rule">
    Set the rule name, select event types, and optionally filter by object key prefix and suffix.
  </Step>
  <Step title="Set the notification target">
    Choose the target service (MNS queue, MNS topic, or Function Compute) and configure the endpoint.
  </Step>
</Steps>

## Filtering rules

| Filter | Example | Matches |
|--------|---------|---------|
| Prefix | `images/` | Objects in the `images/` directory |
| Suffix | `.jpg` | All JPEG files |
| Prefix + Suffix | `uploads/` + `.pdf` | PDF files in the `uploads/` directory |

<Tip>
Create multiple rules with different prefix/suffix filters to route different object types to different processing pipelines.
</Tip>
