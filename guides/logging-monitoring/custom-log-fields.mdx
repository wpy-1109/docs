OSS access logs include a default set of fields. For more precise tracking and analysis, you can configure custom log fields to record specific request headers and query parameters in your logs.

## Use cases

- Track user sessions by recording a custom `X-Session-ID` header.
- Identify traffic sources by logging a `X-Source-App` header.
- Capture custom query parameters for business-specific analytics.

## Configure custom log fields

<Steps>
  <Step title="Open log field settings">
    Navigate to your bucket, choose **Logging > Custom Log Fields**.
  </Step>
  <Step title="Add request headers">
    Specify the HTTP request headers you want to record. For example, `X-Session-ID` or `X-Forwarded-For`.
  </Step>
  <Step title="Add query parameters">
    Specify query string parameters to capture. For example, `source` or `campaign`.
  </Step>
</Steps>

<Note>
Custom log fields are only available when [real-time log query](/guides/logging-monitoring/real-time-log-query) is enabled. The custom fields appear as additional columns in your SLS logstore.
</Note>

## How it works

When a client sends a request with the configured headers or query parameters, OSS includes their values in the access log entry. If a configured field is not present in a request, the log entry records a hyphen (`-`) for that field.
