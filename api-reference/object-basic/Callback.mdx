---
title: 'Callback'
description: 'Configure upload callbacks to notify your application server'
---

You can include callback parameters in PutObject, PostObject, and CompleteMultipartUpload requests. After the upload completes, OSS sends a callback to your server with the upload result.

## Callback parameters

Include the callback configuration in the `x-oss-callback` header (Base64-encoded JSON):

```json
{
  "callbackUrl": "https://your-server.com/callback",
  "callbackHost": "your-server.com",
  "callbackBody": "bucket=${bucket}&object=${object}&size=${size}&etag=${etag}",
  "callbackBodyType": "application/x-www-form-urlencoded"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `callbackUrl` | String | Yes | URL OSS sends the callback to. Multiple URLs separated by `;`. |
| `callbackHost` | String | No | Host header in the callback request. |
| `callbackBody` | String | Yes | Callback body template with OSS variables. |
| `callbackBodyType` | String | No | `application/x-www-form-urlencoded` (default) or `application/json`. |

### System variables

| Variable | Description |
|----------|-------------|
| `${bucket}` | Bucket name |
| `${object}` | Object key |
| `${etag}` | Object ETag |
| `${size}` | Object size |
| `${mimeType}` | Content-Type |
| `${imageInfo.height}` | Image height (images only) |
| `${imageInfo.width}` | Image width (images only) |

## Callback flow

1. Client uploads object to OSS with callback parameters
2. OSS stores the object
3. OSS sends HTTP POST to your callback URL
4. Your server processes and returns HTTP 200
5. OSS returns the upload result to the client

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `CallbackFailed` | 203 | OSS uploaded the object but the callback failed. |
| `InvalidArgument` | 400 | Callback parameters are malformed. |
