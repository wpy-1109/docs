---
title: 'CopyObject'
description: 'Copy an object within or across buckets'
---

Creates a copy of an object. The source and destination can be the same or different buckets (within the same region). Maximum copyable object size is 5 GB. For larger objects, use [UploadPartCopy](/api-reference/multipart-upload/UploadPartCopy).

## Request

```
PUT /{DestObjectName} HTTP/1.1
Host: {DestBucketName}.oss-{region}.aliyuncs.com
x-oss-copy-source: /{SourceBucketName}/{SourceObjectName}
```

### Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `x-oss-copy-source` | String | Yes | Source object path: `/{bucket}/{key}`. |
| `x-oss-copy-source-if-match` | String | No | Copy only if source ETag matches. |
| `x-oss-copy-source-if-none-match` | String | No | Copy only if source ETag differs. |
| `x-oss-copy-source-if-modified-since` | String | No | Copy only if source modified after date. |
| `x-oss-copy-source-if-unmodified-since` | String | No | Copy only if source not modified after date. |
| `x-oss-metadata-directive` | String | No | `COPY` (default, copy source metadata) or `REPLACE` (use request headers). |
| `x-oss-server-side-encryption` | String | No | Encryption for the destination object. |
| `x-oss-storage-class` | String | No | Storage class for the destination object. |
| `x-oss-tagging-directive` | String | No | `Copy` (default) or `Replace`. |

## Response

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CopyObjectResult>
  <LastModified>2025-04-17T03:15:40.000Z</LastModified>
  <ETag>"D41D8CD98F00B204E9800998ECF8****"</ETag>
</CopyObjectResult>
```

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `NoSuchKey` | 404 | Source object does not exist. |
| `EntityTooLarge` | 400 | Source object exceeds 5 GB. Use UploadPartCopy. |
