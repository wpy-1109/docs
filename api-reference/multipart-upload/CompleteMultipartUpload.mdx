---
title: 'CompleteMultipartUpload'
description: 'Complete a multipart upload by assembling parts'
---


## Request

```
POST /{ObjectName}?uploadId={UploadId} HTTP/1.1
Host: {BucketName}.oss-{region}.aliyuncs.com
```

### Request body

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CompleteMultipartUpload>
  <Part>
    <PartNumber>1</PartNumber>
    <ETag>"7265F4D211B56873A381D321F586E4A9"</ETag>
  </Part>
  <Part>
    <PartNumber>2</PartNumber>
    <ETag>"A381D321F586E4A97265F4D211B56873"</ETag>
  </Part>
</CompleteMultipartUpload>
```

Parts must be listed in ascending order by part number.

## Response

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CompleteMultipartUploadResult>
  <Location>https://my-bucket.oss-cn-hangzhou.aliyuncs.com/large-file.bin</Location>
  <Bucket>my-bucket</Bucket>
  <Key>large-file.bin</Key>
  <ETag>"D41D8CD98F00B204E9800998ECF8****-2"</ETag>
</CompleteMultipartUploadResult>
```

<Note>
The ETag of a multipart-uploaded object has the format `"{MD5}-{PartCount}"`, which differs from single-upload ETags.
</Note>

## Error codes

| Error Code | HTTP Status | Description |
|------------|-------------|-------------|
| `NoSuchUpload` | 404 | The upload ID does not exist. |
| `InvalidPart` | 400 | A part ETag does not match. |
| `InvalidPartOrder` | 400 | Parts are not in ascending order. |
| `EntityTooSmall` | 400 | A non-last part is smaller than 100 KB. |
