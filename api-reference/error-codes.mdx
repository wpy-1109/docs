When an API request fails, OSS returns an HTTP 4xx or 5xx status code with an XML error response body.

## Error response format

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Error>
  <Code>NoSuchBucket</Code>
  <Message>The specified bucket does not exist.</Message>
  <RequestId>5C3D9175B6FC201293AD****</RequestId>
  <HostId>examplebucket.oss-cn-hangzhou.aliyuncs.com</HostId>
</Error>
```

| Field | Description |
|-------|-------------|
| `Code` | Machine-readable error code |
| `Message` | Human-readable description |
| `RequestId` | Unique request identifier. Always include this when contacting support. |
| `HostId` | The endpoint that processed the request |

## 4xx client errors

| HTTP Status | Error Code | Description |
|-------------|------------|-------------|
| 400 | `InvalidBucketName` | Bucket name violates naming rules (3-63 chars, lowercase, numbers, hyphens). |
| 400 | `InvalidObjectName` | Object key is invalid, empty, or exceeds 1023 bytes. |
| 400 | `TooManyBuckets` | Exceeded maximum bucket limit per account (default: 100). |
| 400 | `InvalidArgument` | A request parameter has an invalid value. |
| 400 | `MalformedXML` | Request body XML is not well-formed. |
| 400 | `InvalidDigest` | The `Content-MD5` header does not match the body. |
| 400 | `IncompleteBody` | Request body size does not match `Content-Length`. |
| 400 | `FileImmutable` | Object is protected by a WORM retention policy. |
| 400 | `EntityTooLarge` | Object exceeds the 5 GB single-upload limit. Use multipart upload. |
| 400 | `EntityTooSmall` | Multipart upload part is smaller than 100 KB (except the last part). |
| 400 | `InvalidEncryptionAlgorithmError` | Invalid value for `x-oss-server-side-encryption`. Use `AES256`, `KMS`, or `SM4`. |
| 400 | `RequestTimeout` | `Content-Length` was specified but the body was not sent or is incomplete. |
| 400 | `MissingContentLength` | Missing `Content-Length` header and request does not use chunked encoding. |
| 403 | `AccessDenied` | Authorization failed. Check RAM policy, bucket policy, and ACL. |
| 403 | `InvalidAccessKeyId` | The AccessKey ID does not exist or is inactive. |
| 403 | `SignatureDoesNotMatch` | The request signature does not match. Verify signing process. |
| 403 | `RequestTimeTooSkewed` | Client clock differs from server by more than 15 minutes. Sync with NTP. |
| 403 | `InvalidObjectState` | Object storage class does not allow the operation (e.g., reading an unrestored Archive object). |
| 404 | `NoSuchBucket` | The specified bucket does not exist. |
| 404 | `NoSuchKey` | The specified object does not exist. |
| 404 | `NoSuchUpload` | The multipart upload ID does not exist. It may have been aborted or completed. |
| 405 | `MethodNotAllowed` | The HTTP method is not supported for this resource. |
| 409 | `BucketAlreadyExists` | A bucket with this name already exists (globally unique). |
| 409 | `BucketNotEmpty` | Cannot delete a bucket that contains objects or incomplete multipart uploads. |
| 409 | `ObjectNotAppendable` | The object was not created with AppendObject and cannot be appended to. |
| 409 | `PositionNotEqualToLength` | The append position does not match the current object size. |
| 412 | `PreconditionFailed` | A condition in `If-Match` or `If-Unmodified-Since` was not met. |
| 416 | `InvalidRange` | The requested byte range is not satisfiable. |

## 5xx server errors

| HTTP Status | Error Code | Description |
|-------------|------------|-------------|
| 500 | `InternalError` | An internal server error occurred. Retry with exponential backoff. |
| 503 | `ServiceUnavailable` | The service is temporarily unavailable. Retry with exponential backoff. |
| 503 | `SlowDown` | Request rate limit exceeded. Reduce request frequency and implement backoff. |
| 503 | `Throttling` | Request throttled. Distribute requests across buckets or prefixes. |

## Retry strategy

For 5xx errors and `429` responses, implement exponential backoff with jitter:

```python
import time
import random

def retry_with_backoff(func, max_retries=5):
    for attempt in range(max_retries):
        try:
            return func()
        except ServerError as e:
            if attempt == max_retries - 1:
                raise
            wait = min(2 ** attempt + random.uniform(0, 1), 30)
            time.sleep(wait)
```

<Warning>
Do not retry 4xx errors without fixing the request. These indicate client-side issues that will not resolve with retries.
</Warning>

## Troubleshooting common errors

### AccessDenied (403)

1. Verify the AccessKey ID and Secret are correct and active
2. Check the RAM policy grants the required action on the target resource
3. Review bucket policy for explicit deny rules
4. For STS credentials, confirm the token has not expired
5. Check if Block Public Access is enabled

### SignatureDoesNotMatch (403)

1. Verify the canonical request follows the exact format
2. Ensure all parameters are correctly URL-encoded
3. Confirm you are using Signature V4 (not V1)
4. Check that no proxy or CDN is modifying request headers
5. Verify the signing region matches the endpoint region

### NoSuchKey (404)

1. Verify the object key is correct (case-sensitive)
2. Check if lifecycle rules deleted the object
3. If versioning is enabled, check if the object was deleted (has a delete marker)
4. Confirm you are accessing the correct bucket and region
