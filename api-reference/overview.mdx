Alibaba Cloud Object Storage Service (OSS) provides a RESTful API that enables programmatic access to buckets and objects. All API operations are performed via standard HTTP requests to OSS endpoints.

## Base URL

OSS API requests are sent to region-specific endpoints:

```
https://{BucketName}.oss-{region}.aliyuncs.com
```

For example, a bucket named `my-bucket` in the China (Hangzhou) region uses:

```
https://my-bucket.oss-cn-hangzhou.aliyuncs.com
```

<Note>
Use the internal endpoint (`oss-{region}-internal.aliyuncs.com`) for requests from ECS instances in the same region to avoid internet traffic fees.
</Note>

## Authentication

All requests (except anonymous access) must include a signature in the `Authorization` header. OSS uses the **Signature Version 4** algorithm based on HMAC-SHA256.

```
Authorization: OSS4-HMAC-SHA256 Credential={AccessKeyId}/{Date}/{Region}/oss/aliyun_v4_request,
  AdditionalHeaders={headers},
  Signature={signature}
```

See [Signature V4](/api-reference/signature-v4) for the complete signing process.

## Making your first API call

This example creates a bucket using the [PutBucket](/api-reference/bucket-basic/PutBucket) API.

<Steps>
  <Step title="Obtain credentials">
    Get your AccessKey ID and AccessKey Secret from the [Alibaba Cloud console](https://ram.console.aliyun.com/manage/ak). For production use, create a RAM user with appropriate permissions.
  </Step>
  <Step title="Construct the request">
    ```http
    PUT / HTTP/1.1
    Host: my-bucket.oss-cn-hangzhou.aliyuncs.com
    Date: Thu, 17 Apr 2025 03:15:40 GMT
    x-oss-acl: private
    Authorization: OSS4-HMAC-SHA256 Credential=LTAI****/20250417/cn-hangzhou/oss/aliyun_v4_request,Signature=a7c355****

    <?xml version="1.0" encoding="UTF-8"?>
    <CreateBucketConfiguration>
        <StorageClass>Standard</StorageClass>
        <DataRedundancyType>LRS</DataRedundancyType>
    </CreateBucketConfiguration>
    ```
  </Step>
  <Step title="Handle the response">
    **Success** (HTTP 200):
    ```http
    HTTP/1.1 200 OK
    x-oss-request-id: 534B371674E88A4D8906****
    Date: Thu, 17 Apr 2025 03:15:40 GMT
    Content-Length: 0
    Location: /my-bucket
    ```

    **Error** (HTTP 4xx/5xx):
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <Error>
      <Code>BucketAlreadyExists</Code>
      <Message>The requested bucket name is not available.</Message>
      <RequestId>534B371674E88A4D8906****</RequestId>
      <HostId>my-bucket.oss-cn-hangzhou.aliyuncs.com</HostId>
    </Error>
    ```
  </Step>
</Steps>

## API categories

| Category | Description | Key Operations |
|----------|-------------|----------------|
| [Service Operations](/api-reference/service-operations/ListBuckets) | Account-level operations | ListBuckets, DescribeRegions |
| [Bucket - Basic](/api-reference/bucket-basic/PutBucket) | Create, query, delete buckets | PutBucket, GetBucketInfo, DeleteBucket |
| [Bucket - ACL & Policy](/api-reference/bucket-acl-policy/PutBucketAcl) | Access control and policies | PutBucketAcl, PutBucketPolicy |
| [Bucket - Configuration](/api-reference/bucket-encryption/PutBucketEncryption) | Encryption, tags, transfer | PutBucketEncryption, PutBucketTags |
| [Bucket - Lifecycle](/api-reference/bucket-lifecycle/PutBucketLifecycle) | Lifecycle and versioning | PutBucketLifecycle, PutBucketVersioning |
| [Object - Basic](/api-reference/object-basic/PutObject) | Upload, download, manage objects | PutObject, GetObject, DeleteObject |
| [Multipart Upload](/api-reference/multipart-upload/InitiateMultipartUpload) | Large file uploads | InitiateMultipartUpload, UploadPart |
| [Access Points](/api-reference/access-points/CreateAccessPoint) | Named network endpoints | CreateAccessPoint, PutAccessPointPolicy |

## Request ID

Every OSS response includes an `x-oss-request-id` header. Save this value for troubleshooting -- it is required when contacting support.

## Rate limits

OSS applies per-account and per-bucket rate limits. If you receive HTTP 503 with error code `SlowDown` or `Throttling`, implement exponential backoff with jitter. See [Limits](/get-started/concepts/limits) for specific thresholds.
