A **region** is the physical location of an Alibaba Cloud data center. An **endpoint** is the domain name used to access OSS in that region. Choosing the right region and endpoint affects performance, cost, and compliance.

## Choosing a region

Consider these factors when selecting a region:

| Factor | Recommendation |
|--------|---------------|
| **Geographic proximity** | Choose the region closest to your users for lower latency. |
| **Internal network access** | Place buckets in the same region as your ECS instances to use internal endpoints (no public bandwidth charges, lower latency). |
| **Cost** | Prices vary by region. Compare pricing at [OSS Pricing](https://www.alibabacloud.com/product/oss/pricing). |
| **Compliance** | Select a region that meets your data residency and compliance requirements. |
| **Feature availability** | New features may be available in specific regions first. Check [Release Notes](/resources/release-notes). |

<Tip>
Test network latency to different regions using `curl`:

```bash
# Test latency to China (Hangzhou)
curl -o /dev/null -s -w "Connect: %{time_connect}s\nTTFB: %{time_starttransfer}s\nTotal: %{time_total}s\n" \
  "https://oss-cn-hangzhou.aliyuncs.com"
```
</Tip>

## Endpoint formats

Each region provides multiple endpoint types:

| Endpoint Type | Format | Use Case |
|--------------|--------|----------|
| **Public** | `oss-{region-id}.aliyuncs.com` | Access from the internet |
| **Internal** | `oss-{region-id}-internal.aliyuncs.com` | Access from ECS/services in the same region (no bandwidth charges) |
| **Dual-stack** | `{region-id}.oss.aliyuncs.com` | IPv4 and IPv6 support (select China and Europe regions only) |

**Example for China (Hangzhou)**:
- Public: `oss-cn-hangzhou.aliyuncs.com`
- Internal: `oss-cn-hangzhou-internal.aliyuncs.com`
- Dual-stack: `cn-hangzhou.oss.aliyuncs.com`

### Bucket-level endpoints

To access a specific bucket, prepend the bucket name to the endpoint:

```
https://{bucket-name}.oss-{region-id}.aliyuncs.com
```

For example: `https://my-bucket.oss-cn-hangzhou.aliyuncs.com`

## Important: custom domain requirement for Chinese mainland regions

<Warning>
Starting March 20, 2025, new OSS users must use a custom domain name (CNAME) to perform data API operations on OSS buckets in Chinese mainland regions. Default public endpoints are restricted for data operations. Existing users are not affected. If you access data via HTTPS, you must bind a valid SSL certificate to your custom domain. For details, see [Custom Domain Names](/guides/networking/custom-domain-names).
</Warning>

## Internal vs. public endpoints

<CardGroup cols={2}>
  <Card title="Public endpoint" icon="globe">
    Use for access from the internet or outside of Alibaba Cloud. Traffic incurs bandwidth charges.
  </Card>
  <Card title="Internal endpoint" icon="lock">
    Use from ECS or other Alibaba Cloud services in the same region. No bandwidth charges and lower latency.
  </Card>
</CardGroup>

<Warning>
Always use internal endpoints when accessing OSS from ECS instances in the same region. This avoids unnecessary bandwidth costs and provides more stable connections.
</Warning>
