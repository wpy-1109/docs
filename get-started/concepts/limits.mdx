# Limits and Quotas

This page lists the service limits, quotas, and performance metrics for OSS. Understanding these limits helps you design applications that work within OSS boundaries.

## Bucket limits

| Limit | Value | Adjustable? |
|-------|-------|-------------|
| Maximum buckets per region (per account) | 100 | Yes, via [Quota Center](https://quotas.console.alibabacloud.com/products/oss/quotas) |
| Maximum capacity per bucket | Unlimited | N/A |
| Lifecycle rules per bucket | 1,000 | No |
| Domain names per bucket | 100 | No |
| Back-to-origin rules per bucket | 20 | No |
| Image styles per bucket | 50 | No |

<Note>
After a bucket is created, you cannot change its name, region, or storage class.
</Note>

## Object limits

| Limit | Value |
|-------|-------|
| Maximum size (simple upload, form upload, append upload) | 5 GB |
| Maximum size (multipart upload) | 48.8 TB |
| Maximum objects per delete request (console) | 100 |
| Maximum objects per delete request (SDK) | 1,000 |
| Maximum objects per delete request (ossutil) | Unlimited |
| Object rename in console | Objects up to 1 GB only |

<Warning>
By default, uploading an object with the same key as an existing object overwrites the existing one. Enable versioning or use the `x-oss-forbid-overwrite` header to prevent accidental overwrites.
</Warning>

## Bandwidth limits

The following table shows the maximum bandwidth per Alibaba Cloud account per region.

| Region | Total Download (Internal + Public) | Public Download | Total Upload (Internal + Public) | Public Upload |
|--------|-----------------------------------|-----------------|----------------------------------|---------------|
| China (Shanghai) | 100 Gbps | 10 Gbps | 20 Gbps | 10 Gbps |
| China (Shenzhen) | 100 Gbps | 10 Gbps | 20 Gbps | 10 Gbps |
| China (Beijing) | 100 Gbps | 10 Gbps | 20 Gbps | 10 Gbps |
| China (Hangzhou) | 100 Gbps | 20 Gbps | 20 Gbps | 20 Gbps |
| Singapore | 100 Gbps | 5 Gbps | 20 Gbps | 5 Gbps |
| China (Zhangjiakou) | 20 Gbps | Shared | 20 Gbps | Shared |
| Other China regions | 10 Gbps | Shared | 10 Gbps | Shared |
| Other global regions | 5 Gbps | Shared | 5 Gbps | Shared |

When bandwidth limits are reached, requests are throttled. The response header `x-oss-qos-delay-time` indicates the throttling duration in milliseconds.

## QPS (Queries Per Second)

| Read/Write Mode | Maximum QPS |
|-----------------|-------------|
| Non-sequential read/write | 10,000 |
| Sequential read/write | 2,000 |

<Tip>
To maximize QPS, use random or hashed key prefixes instead of sequential ones (e.g., timestamps). Sequential prefixes concentrate requests on a single partition.
</Tip>

The maximum total QPS for single-file uploads and downloads is 2,000, and peak bandwidth cannot exceed 10 Gbps for those operations.

## Restore quotas

| Storage Class | Restore Time | Daily Restore Quota (per account, per region) |
|--------------|-------------|----------------------------------------------|
| Archive | ~1 minute | No limit |
| Cold Archive (Expedited) | Within 1 hour | 100-120 TB total across all priorities |
| Cold Archive (Standard) | 2-5 hours | 100-120 TB total across all priorities |
| Cold Archive (Bulk) | 5-12 hours | 100-120 TB total across all priorities |
| Deep Cold Archive (Expedited) | Within 12 hours | 10-15 TB total across both priorities |
| Deep Cold Archive (Standard) | Within 48 hours | 10-15 TB total across both priorities |

Cold Archive supports an average of 500 restore requests per second per account per region. Deep Cold Archive supports an average of 100 restore requests per second.

## Image processing limits

| Limit | Value |
|-------|-------|
| Maximum source image size | 20 MB |
| Maximum source dimensions (rotation) | 4,096 x 4,096 pixels |
| Maximum source dimensions (other operations) | 30,000 x 30,000 pixels |
| Maximum total pixels (source) | 250 million |
| Maximum output dimensions | 16,384 x 16,384 pixels |
| Maximum total pixels (output) | 16,777,216 |

### Image processing throughput

| Region | Throughput | QPS |
|--------|-----------|-----|
| China (Hangzhou), Shanghai, Beijing, Zhangjiakou, Shenzhen | 20 MB/s | 50 |
| Other regions | 2 MB/s | 5 |
