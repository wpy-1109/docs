## Bucket naming rules

| Rule | Requirement |
|------|-------------|
| **Length** | 3–63 characters |
| **Characters** | Lowercase letters (`a-z`), digits (`0-9`), hyphens (`-`) |
| **Start/end** | Must start and end with a letter or digit |
| **Uniqueness** | Globally unique across all of OSS |
| **Immutable** | Cannot be renamed after creation |

**Valid**: `my-app-data`, `project2024-logs`, `cdn-assets-prod`

**Invalid**: `My-Bucket` (uppercase), `-my-bucket` (starts with hyphen), `ab` (too short)

<Tip>
Include project, environment, and region in bucket names: `myapp-prod-cn-hangzhou`.
</Tip>

## Object key naming rules

| Rule | Requirement |
|------|-------------|
| **Encoding** | UTF-8 |
| **Length** | 1–1,023 bytes |
| **Start character** | Must not start with `/` or `\` |
| **Case sensitivity** | Keys are case-sensitive |

### Characters to avoid

| Character | Issue |
|-----------|-------|
| `\` (backslash) | Parsing issues in some tools |
| Control characters (0x00–0x1F) | Misinterpreted by HTTP clients |
| `#` (hash) | Interpreted as URL fragment |
| `?` (question mark) | Interpreted as query string delimiter |

### Recommended key patterns

Use forward slashes to create logical hierarchies:

```
images/profile/user-12345.jpg
logs/2024/03/15/access.log
tenants/acme-corp/uploads/invoice.pdf
```

<Warning>
Avoid sequential key prefixes (timestamps, auto-incrementing IDs) for high-throughput workloads. Sequential prefixes concentrate writes on a single partition. Use random or hashed prefixes instead.
</Warning>

```python
import hashlib

def add_prefix(key):
    prefix = hashlib.md5(key.encode()).hexdigest()[:4]
    return f"{prefix}/{key}"
```

## Service limits

### Bucket limits

| Limit | Value | Adjustable? |
|-------|-------|-------------|
| Buckets per region (per account) | 100 | Yes, via [Quota Center](https://quotas.console.alibabacloud.com/products/oss/quotas) |
| Capacity per bucket | Unlimited | N/A |
| Lifecycle rules per bucket | 1,000 | No |
| Domain names per bucket | 100 | No |
| Back-to-origin rules per bucket | 20 | No |
| Image styles per bucket | 50 | No |

### Object limits

| Limit | Value |
|-------|-------|
| Max size (simple/form/append upload) | 5 GB |
| Max size (multipart upload) | 48.8 TB |
| Objects per delete request (SDK) | 1,000 |

### Bandwidth limits

| Region | Download (total) | Upload (total) |
|--------|-----------------|----------------|
| China (Hangzhou) | 100 Gbps | 20 Gbps |
| China (Shanghai/Beijing/Shenzhen) | 100 Gbps | 20 Gbps |
| Singapore | 100 Gbps | 20 Gbps |
| Other global regions | 5 Gbps | 5 Gbps |

### QPS limits

| Access pattern | Max QPS |
|---------------|---------|
| Non-sequential read/write | 10,000 |
| Sequential read/write | 2,000 |

<Tip>
Use random key prefixes to maximize QPS. Sequential prefixes concentrate requests on a single partition.
</Tip>

### Image processing limits

| Limit | Value |
|-------|-------|
| Max source image size | 20 MB |
| Max source dimensions | 30,000 × 30,000 px |
| Max output dimensions | 16,384 × 16,384 px |
