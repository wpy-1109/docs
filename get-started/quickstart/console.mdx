# Quickstart: OSS Console

<Info>
This quickstart takes approximately 5 minutes to complete.
</Info>

The OSS Console is a web-based interface for managing your buckets and objects. It requires no software installation and is the easiest way to get started with OSS.

## Prerequisites

Before you begin, make sure you have:

- An [Alibaba Cloud account](https://account.alibabacloud.com/register/intl_register.htm). If you do not have one, create an account first.
- [OSS activated](https://www.alibabacloud.com/product/oss) for your account.
- An [AccessKey pair](https://ram.console.alibabacloud.com/manage/ak) created for your account or a RAM user with OSS permissions.

## Steps

<Steps>
  <Step title="Log in to the OSS Console">
    Go to the [OSS Console](https://oss.console.alibabacloud.com/overview) and log in with your Alibaba Cloud account.
  </Step>

  <Step title="Create a bucket">
    1. In the left navigation pane, click **Buckets**, then click **Create Bucket**.
    2. Configure the bucket:
       - **Bucket Name**: Enter a globally unique name (e.g., `my-first-bucket-2024`). The name must be 3--63 characters, lowercase letters, digits, and hyphens only.
       - **Region**: Select the region closest to your users (e.g., China (Hangzhou)).
       - **Storage Class**: Select **Standard** for general-purpose storage.
       - **Redundancy Type**: Select **Locally Redundant Storage (LRS)** for development, or **Zone-Redundant Storage (ZRS)** for production workloads.
    3. Click **OK** to create the bucket.

    <Tip>
    Choose a region close to your application servers for lower latency. If your ECS instances are in the same region, you can use internal endpoints to avoid public bandwidth charges.
    </Tip>
  </Step>

  <Step title="Upload a file">
    1. Click the bucket name to open the bucket details page.
    2. Click the **Objects** tab, then click **Upload**.
    3. In the upload dialog, click **Select Files** and choose a file from your computer.
    4. Click **Upload** to start the upload.

    After the upload completes, the file appears in the object list.
  </Step>

  <Step title="Download or preview the file">
    1. In the object list, find the file you uploaded.
    2. Click the file name to open the object details page.
    3. Click **Download** to save the file to your computer, or click **Open File URL** to preview it in your browser.

    <Note>
    If the bucket ACL is set to **Private** (the default), you need a signed URL to access the file. The console generates a temporary signed URL automatically when you click **Open File URL**.
    </Note>
  </Step>

  <Step title="Clean up (optional)">
    To avoid storage charges, delete the test objects and bucket:

    1. Select the objects you uploaded and click **Delete**.
    2. Go back to the bucket list, select your test bucket, and click **Delete Bucket**.

    <Warning>
    A bucket must be empty before it can be deleted. Delete all objects (and incomplete multipart uploads) first.
    </Warning>
  </Step>
</Steps>
