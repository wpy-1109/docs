# Quickstart: ossutil CLI

<Info>
This quickstart takes approximately 5 minutes to complete.
</Info>

ossutil is a command-line tool for managing OSS buckets and objects. It supports batch operations, scripting, and automation.

## Prerequisites

Before you begin, make sure you have:

- An [Alibaba Cloud account](https://account.alibabacloud.com/register/intl_register.htm) with [OSS activated](https://www.alibabacloud.com/product/oss).
- An [AccessKey pair](https://ram.console.alibabacloud.com/manage/ak) (AccessKey ID and AccessKey Secret).

## Steps

<Steps>
  <Step title="Install ossutil">
    Download and install ossutil for your operating system.

    <Tabs>
      <Tab title="macOS">
        ```bash
        curl -o ossutil https://gosspublic.alicdn.com/ossutil/v2/2.0.3/ossutil-v2.0.3-darwin-amd64.zip
        unzip ossutil-v2.0.3-darwin-amd64.zip
        chmod 755 ossutil
        sudo mv ossutil /usr/local/bin/
        ```
      </Tab>
      <Tab title="Linux">
        ```bash
        curl -o ossutil https://gosspublic.alicdn.com/ossutil/v2/2.0.3/ossutil-v2.0.3-linux-amd64.zip
        unzip ossutil-v2.0.3-linux-amd64.zip
        chmod 755 ossutil
        sudo mv ossutil /usr/local/bin/
        ```
      </Tab>
      <Tab title="Windows">
        Download [ossutil for Windows](https://gosspublic.alicdn.com/ossutil/v2/2.0.3/ossutil-v2.0.3-windows-amd64.zip), extract the ZIP file, and add the directory to your system `PATH`.
      </Tab>
    </Tabs>

    Verify the installation:

    ```bash
    ossutil version
    ```
  </Step>

  <Step title="Configure credentials">
    Run the interactive configuration command:

    ```bash
    ossutil config
    ```

    When prompted, enter:
    - **AccessKey ID**: Your AccessKey ID
    - **AccessKey Secret**: Your AccessKey Secret
    - **Endpoint**: The endpoint for your region (e.g., `oss-cn-hangzhou.aliyuncs.com`)

    <Warning>
    Never hardcode credentials in scripts for production use. Use environment variables or STS temporary credentials instead.
    </Warning>
  </Step>

  <Step title="Create a bucket">
    ```bash
    ossutil mb oss://my-first-bucket-2024 --region cn-hangzhou
    ```

    You should see output confirming the bucket was created.
  </Step>

  <Step title="Upload a file">
    Upload a local file to your bucket:

    ```bash
    # Upload a single file
    ossutil cp local-file.txt oss://my-first-bucket-2024/

    # Upload a directory recursively
    ossutil cp local-dir/ oss://my-first-bucket-2024/backup/ --recursive
    ```
  </Step>

  <Step title="List and download objects">
    ```bash
    # List objects in the bucket
    ossutil ls oss://my-first-bucket-2024/

    # Download an object
    ossutil cp oss://my-first-bucket-2024/local-file.txt downloaded-file.txt
    ```
  </Step>

  <Step title="Clean up (optional)">
    ```bash
    # Delete all objects in the bucket
    ossutil rm oss://my-first-bucket-2024/ --recursive --force

    # Delete the bucket
    ossutil rb oss://my-first-bucket-2024
    ```

    <Warning>
    Deleted objects cannot be recovered unless versioning is enabled. Proceed with caution.
    </Warning>
  </Step>
</Steps>

## Common commands

| Command | Description |
|---------|-------------|
| `ossutil mb oss://bucket-name` | Create a bucket |
| `ossutil rb oss://bucket-name` | Delete an empty bucket |
| `ossutil ls oss://bucket-name/` | List objects in a bucket |
| `ossutil cp source destination` | Copy files (upload, download, or copy between buckets) |
| `ossutil rm oss://bucket-name/key` | Delete an object |
| `ossutil stat oss://bucket-name` | View bucket information |
| `ossutil sign oss://bucket-name/key` | Generate a presigned URL |

For the full command reference, see [ossutil Commands](/tools/ossutil/commands).
